# `apply_behavior.md`

## Summary

- `cmoc apply fork` において、修正点リスト完成後に作業計画を別途立てる案を採用しなかった理由を整理した補足メモ。
- 修正作業の前に高度な計画立案を挟むことのコストと、実装時の乖離リスクをどう見たかをまとめている。
- 抜け漏れ対策は計画分離ではなく、修正点リスト側をよりリッチにする方がよいという判断を述べている。

## Read this when

- `cmoc apply fork` の設計で、修正点リスト完成後に追加の作業計画を立てるべきか迷っているとき。
- 修正作業の前段に高コストな推論ステップを置く是非を検討したいとき。
- `cmoc apply fork` の実装方針として、事前計画よりも修正点リストの充実を優先する理由を確認したいとき。

## Do not read this when

- `cmoc apply fork` の具体的な実装手順やコード配置を知りたいとき。
- `cmoc session fork`、`cmoc session join`、`cmoc eval-oracles` など、別サブコマンドの仕様を調べたいとき。
- `cmoc` 全体の実行時仕様や共通ルールを広く把握したいだけで、この「採用しなかった案」の理由が不要なとき。

# `memory_alternative.md`

## Summary

- cmoc において、AI が生成した改善案や記憶を次回以降の実行に自動注入しない理由を述べている。
- 実行ログ、明示的な仕様、明示的な成果物、`INDEX`、`oracles` を根拠にする方がよい、という方針を説明している。
- `tgbt plan` や `/plan` のような作業計画レビューを採用しなかった理由と、`cmoc eval-oracle` のように人間が `oracles` を書き、AI がレビューするスタイルを採った背景をまとめている。
- `cmoc apply fork` で修正点リストの後に事前計画を立てる案を採用しなかった理由を整理している。

## Read this when

- AI-generated な memory や kaizen を次回以降の実行コンテキストへ自動的に引き継ぐべきか判断したいとき。
- `oracles` とは別に暗黙の仕様レイヤーを増やすことの是非を検討したいとき。
- 作業計画レビューや plan ベースの監督よりも、明示的な `oracles` レビューを優先すべきか迷ったとき。
- `cmoc apply fork` の前後で、事前計画を立てる運用を採るかどうかを検討したいとき。

## Do not read this when

- `cmoc` の具体的な実装コードやテストコードの場所を探したいだけのとき。
- `cmoc` の各サブコマンドの具体的な入出力仕様や実装手順だけを知りたいとき。
- `INDEX.md` の生成形式やメンテナンス規則そのものを確認したいとき。
- memory や plan review ではなく、別の運用ルールや環境設定だけを調べたいとき。

# `working_plan_review.md`

## Summary

- `tgbt plan` や `/plan` のような作業計画レビュー方式を採用しなかった理由を説明する文書。
- 人間と AI が同じ土俵で作業計画を共同レビューするより、人間が `oracles` に want を書き、AI が実装で追従する方針を選ぶ経緯をまとめている。
- 2026/5 時点では、AI 生成の計画や改善案を次回実行へ自動注入する運用を避けるべきだという判断を示している。

## Read this when

- 作業計画レビューをワークフローに入れるかどうか判断したいとき。
- 人間が plan を書いて AI にレビューさせる方式ではなく、`oracles` を人間が編集する方式を採る理由を確認したいとき。
- AI 生成の改善案や暗黙記憶を次回以降の実行に自動反映しない方針の背景を知りたいとき。
- cmoc の人間側と AI 側の役割分担を、設計思想として理解したいとき。

## Do not read this when

- `cmoc apply fork` の具体的な実装手順や入出力仕様だけを調べたいとき。
- `cmoc eval-oracles` や `cmoc session join` など、個別サブコマンドの操作仕様だけを確認したいとき。
- cmoc 自体のコーディング規約、設計規約、テスト規約を調べたいとき。
- `README.md` や `AGENTS.md`、`memo` などの編集可否や運用ルールだけを確認したいとき。
