
# コンソール・ファイル、ログ出力規則

## サブコマンド呼び出しログ

- サブコマンドの呼び出しと１：１で存在するログ
- コンソール・ファイル両方に出力 (tee) する必要がある
- サブコマンド呼び出しログの役割は以下の２つである
    - cmoc の稼働状況を人間が標準出力経由で確認出来ること
    - 過去の cmoc 呼び出しで何が起きたかを、このサブコマンド呼び出しログファイルを起点として辿れること
- 必要な項目例は以下の通り
    - サブコマンドを構成するステップに入ったことを表す通知
    - Codex CLI を呼び出したことを表す通知
    - サブコマンドを構成するステップ（サブステップを含む）別の経過時間
    - サブコマンド全体の経過時間
    - サブコマンド全体の Codex CLI quota 回復待ち時間
    - サブコマンドの戻り値
- サブコマンドの成功・非成功の扱い
    - 「必要な項目」は成功・非成功を問わず、可能な範囲内で出力する
- ステップ開始通知のフォーマット
    - `(i/N) <ステップの短い説明>` とする
    - i : 全ステップ中何ステップ目か？
    - N : 最終ステップ数
- Codex CLI 呼び出し通知のフォーマット
    - 何を目的とした呼び出しであるかを表す短い説明
    - 対応する `codex exec` 呼び出しログファイルの `<repo-root>` を起点とする相対パス
    - `codex` 呼び出しから完了までにかかった時間
    - `codex` が返した戻り値
- 途中経過・作業完了レポートの表示上の要件
    - 「途中経過」と「作業完了レポート」はその境目がわかりやすい用に工夫すること

## `codex exec` 呼び出しログ

- `codex exec` の呼び出しと１：１で対応するログ
- その呼び出しについての、可能な限り詳細な情報を必要とする
- フォーマットは markdown で、その呼び出し自体についての情報は YAML Front Matter に、入出力は本文に記録する

## 時間表示のフォーマット

- 標準出力に流す時間表示のフォーマットは `<hour>h <minute>m <sec>.<msec>s` とする
- `<hour>`, `<minute>`, `<sec>` は 2 桁・スペースパディング・右詰めとする
- `<msec>` は 1 桁・ゼロ表示・小数点第 2 位以降は切り捨てとする
