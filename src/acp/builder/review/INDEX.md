# `oracle`

## Summary
- `cmoc review oracle` におけるレビュー所見処理の AI 呼び出し構築と、その各段階の構造化応答契約を扱う領域。
- 正本仕様断片を読む前提で、新規所見の列挙、所見を支持する理由の追加調査、所見を否定する理由の追加調査、人間へ提示すべきかの採否判定、所見群の重複・矛盾整理を行うための prompt、補助文脈、モデル区分、推論量、読み取り権限、応答契約の接続を確認する入口となる。

## Read this when
- `cmoc review oracle` のレビュー所見処理で、AI にどの役割・目的・補助情報を渡しているかを確認または変更したいとき。
- レビュー対象の正本仕様断片と関連する正本仕様断片を読ませ、新規所見だけを列挙させる prompt や応答契約を確認したいとき。
- 対象所見について、既知理由と重複しない支持理由または否定理由を、正本仕様断片に基づいて列挙させる処理を確認したいとき。
- 支持理由と否定理由を踏まえて、所見を人間へ提示するかどうかを判定する AI 呼び出しや応答契約を確認したいとき。
- 複数のレビュー所見から重複や相互矛盾を整理するための AI 呼び出し、編集判断、応答契約を確認したいとき。
- レビュー所見処理で使うモデル区分、推論量、正本仕様断片の読み取り専用アクセス、共通 prompt 文脈との接続を追いたいとき。

## Do not read this when
- `cmoc review oracle` の CLI 引数解析、サブコマンド配線、実行順序、入出力管理、保存、表示を確認したいとき。
- レビュー対象となる正本仕様断片そのものの内容や、具体的なレビュー根拠を確認したいとき。
- 正本仕様断片と具現化ファイルの基本定義、またはレビュー標準そのものを確認したいとき。
- 共通 prompt 部品、構造化文書の描画、パス解決、AI 呼び出しパラメータ型の汎用実装を確認したいとき。
- レビュー所見以外の応答契約、CLI 出力、設定、永続状態、または他種別レビューの処理を確認したいとき。
- 既に確定した所見をそのまま保存・表示・転送するだけで、AI に列挙、検証、採否判定、整理をさせない処理を扱うとき。
