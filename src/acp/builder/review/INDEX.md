# `oracle`

## Summary
- oracle レビュー用の ACP 呼び出しパラメータ構築と、その応答を受け取る Structured Output 契約をまとめる領域。新規所見の列挙、所見の擁護・反証理由の追加調査、採否判定、重複・矛盾する所見の整理といった、oracle file を根拠にしたレビュー工程ごとの AI 入出力境界を扱う。
- 各工程では、対象所見や既知理由を補助文脈として渡し、oracle 標準・レビュー標準・ファイルアクセス制約・モデル設定・推論強度・schema 接続を組み合わせて、機械処理可能なレビュー結果を返させるための入口になる。

## Read this when
- oracle レビューで、新規所見の検出、所見の妥当性を支持する理由の追加調査、所見への反証理由の追加調査、採否判定、複数所見の整理という各 AI 呼び出しの prompt 構成や呼び出し条件を確認・変更したいとき。
- oracle file だけを根拠として扱わせるレビュー調査エージェントに、対象所見、既知の擁護理由、既知の反論理由、既知所見との差分などをどのように渡すか追いたいとき。
- レビュー所見や理由列挙、採否判定、所見統合の結果を機械処理可能に返す契約を確認し、重大度、根拠、採否理由、削除・置換・統合の意味的境界を実装・検証したいとき。
- oracle レビュー後処理として、重複所見の削除、矛盾する所見の置換、複数所見の統合をどの粒度で表現するか確認したいとき。

## Do not read this when
- oracle file そのものの正本仕様内容、oracle file の定義・管理ルール、またはレビュー対象として読むべき正本仕様断片を判断したいとき。
- oracle レビュー全体の CLI 引数解析、サブコマンド定義、実行順序、所見の保存・表示・集計・通知など、AI 呼び出しパラメータ構築より外側の処理を調べたいとき。
- レビュー以外の ACP 呼び出しパラメータ構築、共通プロンプト部品、または汎用的な JSON Schema 構文だけを確認したいとき。
- oracle file を実際に読んで矛盾・誤字・表記揺れを発見する判断そのものや、所見に基づく oracle file の具体的な編集案を作りたいとき。
