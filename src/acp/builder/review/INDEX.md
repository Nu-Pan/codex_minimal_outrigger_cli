# `oracle`

## Summary
- `cmoc review oracle` のレビュー工程で使う AI 呼び出しパラメータ構築と Structured Output 契約をまとめた領域。正本仕様断片を対象に、新規所見の列挙、所見同士の重複・矛盾整理、所見を支持する理由と否定する理由の追加調査、人間へ提示すべきかの採否判定までを扱う。
- 各工程は、oracle file を読み取り専用の根拠として扱う前提、既知情報との重複排除、推測ではなく正本仕様断片に基づく判断、結果を機械処理できる形で返すための出力契約を結び付ける入口になっている。

## Read this when
- `cmoc review oracle` で、oracle file レビューに関わる AI 呼び出しの prompt、補助文脈、モデル区分、推論量、ファイルアクセス方針、Structured Output schema との接続を確認または変更したいとき。
- レビュー対象の oracle file と関連する oracle file から、新規所見を列挙する処理、既知所見との重複を避ける指示、所見が無い場合の扱いを確認したいとき。
- 収集済みのレビュー所見について、重複・矛盾を削除、置換、統合として整理する AI 出力契約や、その整理用 prompt を確認したいとき。
- 対象所見が妥当である理由、または妥当ではない理由を、既知理由と重複しない oracle file 由来の根拠として追加調査する処理を確認したいとき。
- レビュー所見を人間へ提示するかどうかの採否判定について、判定理由を含む出力契約や、所見本文・支持理由・反対理由を AI 呼び出しへ渡す構成を確認したいとき。

## Do not read this when
- 個別の oracle file の本文をレビューして、具体的な仕様矛盾、表記揺れ、誤字脱字などの所見を探したいとき。この領域はレビュー工程の呼び出し構築と出力契約を扱い、レビュー根拠そのものは対象の正本仕様断片を読む。
- oracle file と realization file の基本定義、責務境界、正本仕様断片としての一般基準を確認したいとき。基本概念や標準を定義する oracle 側の文書へ進む。
- `cmoc review oracle` 全体の CLI 引数解析、サブコマンド配線、実行順序、所見の保存・表示・適用処理を確認したいとき。呼び出し側の command 実装や所見管理側を読む。
- 共通 prompt 部品、構造化文書の描画、パス解決、AI 呼び出しパラメータ型そのものなど、レビュー工程に限定されない共通基盤を確認したいとき。
- レビュー所見以外の Structured Output 契約、一般的な JSON Schema 構文、UI、通知、集計、永続状態の形式を確認したいとき。
