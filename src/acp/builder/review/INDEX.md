# `oracle`

## Summary
- `cmoc review oracle` におけるレビュー担当 AI 呼び出しの入口と、その応答を受け取る Structured Output 契約をまとめる領域。
- oracle file から新規所見を列挙する段階、所見の妥当性を支持・否定する理由を追加調査する段階、所見を人間へ提示するか判定する段階、所見リストの重複や矛盾を整理する段階を扱う。
- 各段階で使うロール、目標、補助文脈、oracle 読み取り前提、モデル区分、推論量、応答スキーマとの対応を確認する入口になる。

## Read this when
- `cmoc review oracle` の所見列挙、所見検証、採否判定、所見整理に使う AI 呼び出しパラメータを確認または変更したいとき。
- oracle review 用プロンプトが、レビュー対象、既知所見、既知の支持理由、既知の否定理由、現状の所見リストをどのように補助文脈として渡すか確認したいとき。
- oracle file を根拠にしたレビュー結果について、重複しない新規所見、新規の支持理由、新規の否定理由、採否判定、整理操作のいずれかの Structured Output 契約を確認したいとき。
- oracle review の各段階で、読み取り専用の oracle アクセス、標準文脈、モデル区分、推論量、応答スキーマ参照の接続を追いたいとき。

## Do not read this when
- `cmoc review oracle` の CLI 引数解析、サブコマンド配線、実行順序、保存、表示、入出力管理を調べたいとき。
- レビュー対象となる個々の oracle file の正本仕様本文や、oracle file 自体の標準・品質基準を読みたいとき。
- 共通プロンプト部品、構造化文書レンダリング、パス解決、AI 呼び出しパラメータ型そのものの実装を確認したいとき。
- oracle review 以外のレビュー種別や、一般的な JSON Schema 構文だけを調べたいとき。
