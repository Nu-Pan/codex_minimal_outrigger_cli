# `enumerate_finding.json`

## Summary
- oracle file のレビューで、既知の関連所見と重複しない新規所見だけを返すための Structured Output schema を定義する。
- 新規所見が無い場合の扱い、所見ごとに必要な判断情報、重大度の境界を固定し、oracle レビュー結果を機械処理できる形に揃える入口となる。

## Read this when
- oracle file レビュー結果を返す処理やプロンプトで、新規所見の列挙形式を確認したいとき。
- レビュー対象 oracle file と関連 oracle file から見つけた問題を、既知の関連所見と区別して出力する契約を確認したいとき。
- 所見の重大度を、明確な仕様矛盾・実装者裁量で解消不能な問題と、誤字脱字・表記揺れなどの単純な表記問題に分ける境界を確認したいとき。
- 新規所見が無い場合に、結果をどのように表現するかを確認したいとき。

## Do not read this when
- oracle file の本文そのものをレビューして具体的な所見を探したいとき。この対象は所見の出力契約であり、レビュー根拠は対象の oracle file や関連 oracle file を読む。
- oracle file と realization file の基本定義や責務境界を確認したいとき。基本概念を定義する oracle 側の文書を読む。
- レビュー所見以外の Structured Output schema、CLI 出力、設定、永続状態の形式を確認したいとき。該当する schema や実装の対象へ進む。
- 既知の関連所見の内容や、その所見が既知扱いされる理由を確認したいとき。この対象は既知所見の一覧や管理規則を保持しない。
