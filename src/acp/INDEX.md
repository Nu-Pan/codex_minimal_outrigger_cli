# `builder`

## Summary
- AI エージェントへ渡す呼び出しパラメータを、用途別の prompt、補助文脈、ファイルアクセス権限、モデル設定、推論強度、Structured Output schema と結び付けて構築する領域。実装適用後の差分要約や所見処理、正本仕様断片レビュー、ルーティング文書エントリー生成、TUI 実行前のパラメータ選定、merge conflict marker 解消依頼など、各サブタスクをエージェント呼び出しへ変換する入口になる。
- 各下位領域は、対象タスクの role・summary・goal・標準文書参照・追加入力の埋め込み・読み書き権限・返却 schema を定義し、実際の CLI 制御や結果保存ではなく、AI に何をどう依頼するかという契約を扱う。

## Read this when
- エージェント呼び出しに渡す完全 prompt の内容、追加文脈の埋め込み方、参照させる標準文書、ファイルアクセスモード、モデル種別、reasoning effort、Structured Output schema の対応関係を確認または変更したいとき。
- 実装適用後の git 差分を人間向けに要約させる処理、ファイル単位で realization file の要修正点を列挙させる処理、検出済み所見を修正担当へ渡す処理の呼び出し条件を追いたいとき。
- 正本仕様断片レビューで、新規所見の列挙、所見を支持または否定する理由の追加調査、所見の採否判定、所見リストの重複・矛盾整理をエージェントへ依頼する内容を確認したいとき。
- ルーティング文書用エントリー生成で、対象本文や関連文書をどのような制約で読ませ、どの schema に従って summary・読む条件・読まない条件を返させるか確認したいとき。
- TUI に入力された元プロンプトから、実行時のファイルアクセス権限や標準文書の参照要否を選定させるための判定 prompt と返却契約を確認したいとき。
- session join 時の merge conflict marker 解消について、対象ファイル一覧、作業範囲、例外的な編集許可、git 操作禁止などをエージェント呼び出しへどう渡すか確認したいとき。
- 各サブタスクの構造化出力が、変更カテゴリ、主要変更パス、所見根拠、仕様要求、観測された実装、修正方針、レビュー理由、採否、整理操作、パラメータ選定理由などをどう要求しているか確認したいとき。

## Do not read this when
- CLI サブコマンドの引数解析、実行順序、分岐制御、結果の保存・表示・集約、git コマンド実行、フォークやセッションの作成・統合そのものを調べたいとき。
- 正本仕様断片、realization file、各種標準文書、レビュー基準、適用レビュー基準、ルーティング文書作成基準の本文そのものを読みたいとき。
- 汎用的な prompt 部品の組み立て、Markdown レンダリング、構造化ドキュメント表現、論理パスから実パスへの解決、エージェント呼び出し型やファイルアクセスモードの基礎定義を調べたいとき。
- 所見抽出やカテゴリ分けの実際の判断基準、個別レビュー対象の内容、git diff 生成方法、変更ファイル抽出アルゴリズム、merge conflict の自動編集アルゴリズムを調べたいとき。
- TUI の画面表示、エディタ入力の取得、コメント除去、入力文字列の strip、対話フローなど、呼び出しパラメータ解決より外側の UI 実装を調べたいとき。
- 生成済みのルーティング文書の妥当性や、特定ディレクトリで実際にどの本文へ進むべきかだけを判断したいとき。

# `prompt_parts`

## Summary
- AI agent に渡すプロンプトを構成する部品群を扱うディレクトリ。ファイルアクセス規則、ルーティング規則、oracle/realization の基本説明、oracle・realization・レビュー・INDEX.md エントリーの標準文書を構造化文書として組み立てる実装がまとまっている。
- 個別の標準文書本文を生成する部品と、それらを依頼内容・アクセスモード・追加プロンプト・有効化フラグに応じて完全な agent 用プロンプトへ集約する入口がある。

## Read this when
- AI agent に渡す最終プロンプトの構成、標準文書の注入順序、または有効化フラグ間の依存関係を確認・変更したいとき。
- ファイル読み書き制約、INDEX.md を使った読み進め方、oracle file と realization file の責務境界を、agent 向けプロンプトとしてどの文面で提示しているか確認したいとき。
- oracle file、realization file、oracle review、oracle-to-realization review、INDEX.md エントリー生成に関する標準文書の生成内容を確認・変更したいとき。
- 標準文書を構成する要求・背景・判断例が、どの構造化文書として agent prompt に組み込まれるかを追いたいとき。

## Do not read this when
- 特定の CLI サブコマンド、状態ファイル、path model、入出力 schema などの個別機能仕様や実装詳細を探しているとき。
- 標準文書を利用する側ではなく、構造化文書型、標準項目型、またはそれらの変換基盤そのものを確認したいとき。
- テスト実装や fixture の内容を確認したいとき。
- INDEX.md エントリーの完成文面だけが必要で、対象本文をすでに直接読める状態にあるとき。
