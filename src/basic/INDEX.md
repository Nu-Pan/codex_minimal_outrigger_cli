# `acp.py`

## Summary
- AI コーディングエージェント呼び出し時に渡す論理パラメータを定義する実装。モデルクラス、reasoning effort、ファイルアクセスモードをバックエンド非依存の列挙として表し、プロンプト本文と Structured Output schema の有無を含む呼び出し単位のデータ構造を提供する。
- バックエンドが受理する具体的なモデル名や権限指定への変換はこの対象ではなく、上位の呼び出し制御と realization 側の解決処理の間で共有される入力仕様の入口になる。

## Read this when
- エージェント呼び出しに渡すモデル選択、reasoning effort、ファイルアクセスモード、プロンプト、Structured Output schema パスの保持形式を確認または変更したいとき。
- cmoc 内部で使う論理的なモデル区分やファイルアクセス権限区分を追加・削除・改名する必要があるとき。
- エージェント呼び出しパラメータを生成する処理、受け取る処理、またはテストで、どのフィールドを必須入力として扱うか確認したいとき。

## Do not read this when
- 具体的なバックエンド名、実モデル名、CLI 引数、外部コマンド呼び出し形式への変換ロジックを確認したいだけのとき。
- プロンプト本文の組み立て方、テンプレート内容、タスク種別ごとの文章生成ルールを調べたいとき。
- Structured Output schema ファイル自体の内容や JSON schema の仕様を確認したいとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン付きパス表記と、実パスとの相互変換を扱う実装。絶対パス、`<cmoc-root>`・`<repo-root>`・`<run-root>`・`<work-root>` 付き相対パスを受け取り、各ルートの探索規則に基づいて現実の絶対パスへ解決する。
- ルートトークンの定義、ルートディレクトリ探索、ルートトークンなし相対パスの拒否、実パスを指定ルートトークン表記へ戻す処理を確認する入口になる。

## Read this when
- cmoc 内でパス文字列や `Path` を実際の絶対パスへ解決する挙動を確認・変更したいとき。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味、探索起点、`.git` ディレクトリまたは `.git` ファイルによる判定条件を確認したいとき。
- ルートトークン付き相対パスを許可し、ルートトークンなし相対パスを禁止する入力検証の扱いを確認したいとき。
- linked worktree や git common dir を含む状況で、main worktree と run 用 worktree の判定がどう行われるかを追いたいとき。
- 実パスを特定のルートトークン基準の表記へ変換する処理や、対象外パスの場合の失敗条件を確認したいとき。

## Do not read this when
- CLI サブコマンドの引数定義、画面出力、終了コードなど、利用者向けコマンド挙動だけを確認したいとき。
- ルートトークンの概念ではなく、個別機能がどの作業ディレクトリでファイルを作成・更新するかという業務ロジックを確認したいとき。
- oracle file と realization file の所有関係や編集責務など、仕様管理上の分類だけを確認したいとき。
- テスト構成、fixture、テストケース追加先を探しているとき。

# `standard.py`

## Summary
- 規範文書を表すデータ構造と、その要求項目を構造化ドキュメントへ変換する処理を定義する。
- 規範の見出し、背景、要求、判断例を保持し、要求ラベルの種類と本文形式を表す入口になる。

## Read this when
- 規範をコード上で表現するためのクラス、必須フィールド、値検証の挙動を確認したいとき。
- 要求項目に使えるラベルや、要求本文をどのような単位で保持するかを確認したいとき。
- 規範オブジェクトから構造化ドキュメントを生成する際の見出し構成や出力されるフィールドを確認したいとき。

## Do not read this when
- 個別の規範そのものの内容や、人間が責任を持つ正本仕様断片を確認したいとき。
- 構造化ドキュメント型そのものの責務、表現形式、描画・整形方法を確認したいとき。
- CLI コマンド、ファイル走査、パスモデル、テスト実行など、規範データ構造の定義と変換以外の実装を調べたいとき。

# `struct_doc.py`

## Summary
- 階層構造を持つ自然言語文書を Markdown に変換するための小さなヘルパーを定義する実装。
- 見出し付きの文書ノード、挿入可能なコードブロック、Markdown レンダリング、連続空行の整理、トリプルクォート文字列向けのインデント正規化を扱う。
- 文書構造を組み立てて、見出し深さを自動で調整した Markdown 文字列を得たい処理の入口になる。

## Read this when
- 自然言語の構造化文書をプログラム上で組み立て、Markdown として出力する処理を確認・変更する場合。
- 見出し階層、子要素の型制約、コードフェンス付きコードブロックの出力形式に関わる挙動を調べる場合。
- 複数行文字列の先頭末尾空行削除、共通インデント解除、連続空行の畳み込みが Markdown 出力にどう影響するか確認する場合。

## Do not read this when
- Markdown の解析、既存 Markdown 文書からの構造抽出、または Markdown 以外の出力形式を探している場合。
- CLI コマンド、ファイル探索、パスモデル、oracle と realization の仕様管理など、構造化文書レンダリング以外の cmoc 全体機能を調べる場合。
- 単なる文字列結合やログ整形など、見出し階層を持つ文書モデルを必要としない出力処理を確認する場合。
