# `acp.py`

## Summary
- AI コーディングエージェント呼び出しに渡す論理パラメータを定義する実装。モデルクラス、Reasoning effort、ファイルアクセスモードをバックエンド非依存の列挙として表し、プロンプト本文と Structured Output schema の有無を含む呼び出し単位のデータ構造をまとめる。
- バックエンドが受理する具体的なモデル名や権限表現ではなく、cmoc 内部で扱う抽象的な呼び出し条件の入口として位置づけられる。

## Read this when
- エージェント呼び出しに必要なパラメータ一式の型や保持項目を確認・変更したいとき。
- モデル品質・コストの抽象区分、Reasoning effort、ファイルアクセスモードの選択肢を確認・変更したいとき。
- Structured Output schema を要求する呼び出しと要求しない呼び出しを、呼び出しパラメータ上でどう表すか確認したいとき。
- プロンプト本文と実行条件をひとまとまりの値として受け渡す箇所を追うとき。

## Do not read this when
- 論理的なモデルクラスやファイルアクセスモードを、特定バックエンドが受理する具体的な値へ解決する処理を探しているとき。
- エージェントプロセスの起動、外部コマンド実行、API 呼び出し、入出力処理そのものを調べたいとき。
- 個別タスクのプロンプト生成内容や Structured Output schema の中身を調べたいとき。
- oracle file と realization file の一般定義や編集責務そのものを確認したいだけのとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン表記と実パス解決の基礎モデルを定義する実装。絶対パス、ルートトークン付き相対パス、ルートトークン単体を現実のパスへ解決し、実パスを指定トークン表記へ戻す責務を持つ。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味と、それぞれを `.git` ディレクトリ、`.git` ファイル、`bin/cmoc`、git common dir などから探索する規則がまとまっている。
- ルートトークンなし相対パスを禁止する入力検証や、各 root 解決関数が共有する親ディレクトリ候補列挙の実装も含む。

## Read this when
- cmoc 内で扱うパス表記のルール、ルートトークンの種類、または各トークンがどの実ディレクトリを指すかを確認したいとき。
- ユーザー入力や内部値として渡されたパスを、絶対パスまたはルートトークン表記へ変換する処理を変更・調査するとき。
- main worktree、linked worktree、現在の worktree の判定方法や、`.git` ディレクトリと `.git` ファイルの扱いを確認したいとき。
- ルートトークンなし相対パスを拒否する挙動、空パスや未対応型に対する例外、git コマンドを使った repository root 推定の挙動を調べるとき。

## Do not read this when
- CLI サブコマンドの入出力、引数解析、実行フローだけを知りたいとき。
- INDEX 生成、oracle file と realization file の分類、または文書ルーティングの規則を調べたいとき。
- 個別機能のファイル操作内容や業務ロジックを確認したいだけで、ルートトークン表記と実パス解決に関心がないとき。
- テスト側で期待される外部挙動を確認したい場合で、実装詳細ではなく検証観点から読み始めたいとき。

# `struct_doc.py`

- 自然言語の階層文書を組み立てるためのデータ構造や、受け付ける子要素の型制約を確認したいとき。
