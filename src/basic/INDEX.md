# `acp.py`

## Summary
- AI コーディングエージェント呼び出しに渡す論理パラメータを定義する実装。モデルクラス、Reasoning effort、ファイルアクセスモードをバックエンド非依存の列挙として表し、プロンプト本文と Structured Output schema の有無を含む呼び出し単位のデータ構造をまとめる。
- バックエンドが受理する具体的なモデル名や権限表現ではなく、cmoc 内部で扱う抽象的な呼び出し条件の入口として位置づけられる。

## Read this when
- エージェント呼び出しに必要なパラメータ一式の型や保持項目を確認・変更したいとき。
- モデル品質・コストの抽象区分、Reasoning effort、ファイルアクセスモードの選択肢を確認・変更したいとき。
- Structured Output schema を要求する呼び出しと要求しない呼び出しを、呼び出しパラメータ上でどう表すか確認したいとき。
- プロンプト本文と実行条件をひとまとまりの値として受け渡す箇所を追うとき。

## Do not read this when
- 論理的なモデルクラスやファイルアクセスモードを、特定バックエンドが受理する具体的な値へ解決する処理を探しているとき。
- エージェントプロセスの起動、外部コマンド実行、API 呼び出し、入出力処理そのものを調べたいとき。
- 個別タスクのプロンプト生成内容や Structured Output schema の中身を調べたいとき。
- oracle file と realization file の一般定義や編集責務そのものを確認したいだけのとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン付きパス表記と実パス解決の実装を担う。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の列挙と、それぞれを現在位置・Git 情報・実装ファイル位置から絶対パスへ解決する処理を提供する。
- 絶対パスやルートトークン付き相対パスを実パスへ変換し、逆に実パスを指定したルートトークン表記へ戻すための入口になる。

## Read this when
- ルートトークンの種類、意味、追加・変更、またはトークン付きパス表記の許可条件を確認したいとき。
- ユーザー入力や内部処理で扱うパスを、絶対パスまたはルートトークン付き表記から実在する絶対パスへ変換する挙動を調べるとき。
- main worktree、linked worktree、現在の worktree のどれを指すべきかという判定ロジックを確認・変更するとき。
- 実パスが特定のルート配下にあるかを検証し、ルートトークン付き表記へ変換する処理を扱うとき。
- `.git` ディレクトリ、`.git` ファイル、Git の common dir を使ったルート探索の失敗条件や例外を確認するとき。

## Do not read this when
- 個別サブコマンドの CLI 引数、出力形式、実行フローを調べたいだけのとき。
- ファイル内容の読み書き、INDEX 生成、oracle と realization の分類など、パス解決後の業務ロジックを調べたいとき。
- テストで期待される外部挙動だけを確認したい場合で、ルートトークン解決の内部実装まで不要なとき。
- Python の `Path` 操作一般や Git worktree 一般の使い方を知りたいだけで、cmoc 固有のルートトークン規則に関係しないとき。

# `struct_doc.py`
