# `acp.py`

## Summary
- AI コーディングエージェント呼び出し時に渡す論理パラメータを定義する実装。モデルクラス、reasoning effort、ファイルアクセスモードをバックエンド非依存の列挙として表し、プロンプト本文と Structured Output schema の有無を含む呼び出し単位のデータ構造を提供する。
- バックエンドが受理する具体的なモデル名や権限指定への変換はこの対象ではなく、上位の呼び出し制御と realization 側の解決処理の間で共有される入力仕様の入口になる。

## Read this when
- エージェント呼び出しに渡すモデル選択、reasoning effort、ファイルアクセスモード、プロンプト、Structured Output schema パスの保持形式を確認または変更したいとき。
- cmoc 内部で使う論理的なモデル区分やファイルアクセス権限区分を追加・削除・改名する必要があるとき。
- エージェント呼び出しパラメータを生成する処理、受け取る処理、またはテストで、どのフィールドを必須入力として扱うか確認したいとき。

## Do not read this when
- 具体的なバックエンド名、実モデル名、CLI 引数、外部コマンド呼び出し形式への変換ロジックを確認したいだけのとき。
- プロンプト本文の組み立て方、テンプレート内容、タスク種別ごとの文章生成ルールを調べたいとき。
- Structured Output schema ファイル自体の内容や JSON schema の仕様を確認したいとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン表記と実パス解決の実装を扱う。ルートトークンの種類、トークン付きパスから絶対パスへの変換、絶対パスからトークン付き表記への変換、各ルートを探索するための候補列挙をまとめている。
- パス表記の禁止事項として、ルートトークンを持たない相対パスを拒否する責務を持つ。`<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味や、git worktree 構成に応じた解決方針を確認する入口になる。

## Read this when
- cmoc 内でファイル・ディレクトリパスをルートトークン付き表記として扱う処理を確認・変更する時。
- ルートトークンの追加、意味変更、または `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の解決条件を確認・変更する時。
- 絶対パスとトークン付きパスの相互変換、ルートトークンなし相対パスの拒否、空パスや型不正時の例外挙動を確認する時。
- git linked worktree 内外で repo root、run root、work root がどのように判定されるかを調べる時。

## Do not read this when
- 個別コマンドの入出力、サブコマンド制御、永続状態、またはユーザー向け CLI 挙動を調べたいだけの時。
- 特定ファイルを実際に読み書きする処理や、ファイル内容の解析・生成ロジックを調べたい時。
- テスト側で期待されるパス解決の具体ケースだけを確認したい場合で、対応するテストファイルを直接読む方が適切な時。

# `standard.py`

## Summary
- 規範文書を表すデータ構造と、その規範を構造化ドキュメントへ変換する処理を定義する実装。規範の見出し、背景、要求、判断例を保持し、要求には必須・禁止・推奨・許容のラベル付き本文を持たせる。
- 規範本文の必須要素を検証し、不正な背景・要求・判断例を拒否するため、規範生成の入力形式や出力構成を確認する入口になる。

## Read this when
- 規範を表すオブジェクトのフィールド、検証条件、保持される本文要素を確認したいとき。
- 必須・禁止・推奨・許容のラベルを持つ要求項目の型や責務を確認したいとき。
- 規範データが背景・要求・判断例の構造化ドキュメントへどのように変換されるかを確認したいとき。
- 規範文書生成まわりで、空リストを許す項目と拒否する項目、または不正入力時の例外条件を調べたいとき。

## Do not read this when
- 個別の規範本文そのものの内容や、人間が管理する正本仕様断片を確認したいだけのとき。
- 構造化ドキュメント一般の表現、整形、出力仕様を確認したいとき。
- 規範以外の文書種別や、CLI コマンド、パスモデル、作業ツリーの挙動を調べたいとき。
- 実際に生成済みの規範文書やテスト期待値を確認したいとき。

# `struct_doc.py`

## Summary
- 階層構造を持つ自然言語文書を Markdown に変換するための基本部品を定義する。見出し付きの文書ノード、埋め込み用コードブロック、Markdown レンダリング、三重引用文字列由来の本文正規化、連続空行の圧縮を扱う。
- 文書構造をオブジェクトとして組み立て、見出し深さを再帰的に計算して Markdown 文字列へ出力したい処理から参照される入口である。

## Read this when
- 階層化された説明文やレポートを Markdown 見出し付きで生成する処理を確認・変更するとき。
- 本文文字列やコードブロックを構造化文書の子要素として扱う仕様、型制約、例外条件を確認するとき。
- Markdown 出力時の見出し深さ、コードフェンス、空行圧縮、三重引用文字列のインデント除去の挙動を確認するとき。

## Do not read this when
- 既存 Markdown ファイルの内容解析や見出し抽出を調べたいとき。この対象は Markdown の読み取りではなく生成を扱う。
- CLI コマンド、入出力先、ファイル探索、永続状態など、生成された Markdown をどこで使うかを調べたいとき。呼び出し側の処理を直接読む方がよい。
- 単なる文字列整形全般やパス表記の定義を調べたいとき。この対象で扱う正規化は構造化文書レンダリングに付随する本文整形に限られる。
