# `acp.py`

## Summary
- AI コーディングエージェント呼び出し時に渡す論理パラメータを定義する実装。モデルクラス、reasoning effort、ファイルアクセスモードをバックエンド非依存の列挙として表し、プロンプト本文と Structured Output schema の有無を含む呼び出し単位のデータ構造を提供する。
- バックエンドが受理する具体的なモデル名や権限指定への変換はこの対象ではなく、上位の呼び出し制御と realization 側の解決処理の間で共有される入力仕様の入口になる。

## Read this when
- エージェント呼び出しに渡すモデル選択、reasoning effort、ファイルアクセスモード、プロンプト、Structured Output schema パスの保持形式を確認または変更したいとき。
- cmoc 内部で使う論理的なモデル区分やファイルアクセス権限区分を追加・削除・改名する必要があるとき。
- エージェント呼び出しパラメータを生成する処理、受け取る処理、またはテストで、どのフィールドを必須入力として扱うか確認したいとき。

## Do not read this when
- 具体的なバックエンド名、実モデル名、CLI 引数、外部コマンド呼び出し形式への変換ロジックを確認したいだけのとき。
- プロンプト本文の組み立て方、テンプレート内容、タスク種別ごとの文章生成ルールを調べたいとき。
- Structured Output schema ファイル自体の内容や JSON schema の仕様を確認したいとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン表記と実パス解決の実装を担うモジュール。絶対パス、ルートトークン付き相対パス、RootToken を現実の絶対パスへ変換し、実パスを指定したルートトークン表記へ戻す責務を持つ。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味と探索方法をコード上で定義し、`.git` ディレクトリ・ファイルや `bin/cmoc` を手がかりに各 root を解決する。
- root token を持たない相対パスを拒否するなど、cmoc 内で許可されるパス表記の境界を実装している。

## Read this when
- cmoc 内で `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の定義や探索条件を確認・変更したいとき。
- root token 表記から実際の絶対パスへの変換、または実パスから root token 表記への変換の挙動を調べたいとき。
- root token を持たない相対パス、空パス、対象 root 外の実パスなど、パス解決時の例外条件を確認したいとき。
- git worktree、main worktree、linked worktree の位置関係に依存する root 解決処理を調べたいとき。

## Do not read this when
- 個別コマンドの CLI 引数、出力 schema、サブコマンド実行フローだけを調べたいとき。
- ファイル内容の読み書き、INDEX.md 生成、oracle file と realization file の分類など、root token 解決そのものではない上位処理を調べたいとき。
- パス表記の正本仕様だけを確認したい場合で、実装上の探索手順や例外挙動までは不要なとき。

# `standard.py`

## Summary
- 規範文書を表すデータ構造と、その規範を構造化文書へ変換する入口を定義する実装。
- 規範の見出し、背景、要求、判断例を保持し、背景・要求・判断例が期待されるリスト内容であることを検証する。
- 要求項目は必須・禁止・推奨・許容のラベルと短い本文で表し、変換時には背景、要求、必要に応じて判断例のセクションへ整形される。

## Read this when
- 規範文書の実装上の表現、保持する本文要素、または入力検証条件を確認・変更したいとき。
- 要求項目のラベル体系や、要求本文をどのような単位で扱うかを確認・変更したいとき。
- 規範を構造化文書へ変換する際のセクション構成や箇条書き整形を確認・変更したいとき。
- 規範が対象とする範囲や、背景・要求・判断例の責務境界を実装から確認したいとき。

## Do not read this when
- 個別の規範そのものの内容、要求文、判断例を確認・変更したいだけのとき。
- 構造化文書クラス自体のデータ構造やレンダリング規則を確認・変更したいとき。
- ルーティング文書のエントリー形式や生成規則だけを確認したいとき。
- 作業対象が規範文書の表現・検証・変換に関係しない CLI 挙動、状態管理、ファイル操作、テスト実行であるとき。

# `struct_doc.py`

## Summary
- 階層構造を持つ自然言語文書を Markdown に変換するための基本部品を定義する。見出し付きの文書ノード、埋め込み用コードブロック、Markdown レンダリング、三重引用文字列由来の本文正規化、連続空行の圧縮を扱う。
- 文書構造をオブジェクトとして組み立て、見出し深さを再帰的に計算して Markdown 文字列へ出力したい処理から参照される入口である。

## Read this when
- 階層化された説明文やレポートを Markdown 見出し付きで生成する処理を確認・変更するとき。
- 本文文字列やコードブロックを構造化文書の子要素として扱う仕様、型制約、例外条件を確認するとき。
- Markdown 出力時の見出し深さ、コードフェンス、空行圧縮、三重引用文字列のインデント除去の挙動を確認するとき。

## Do not read this when
- 既存 Markdown ファイルの内容解析や見出し抽出を調べたいとき。この対象は Markdown の読み取りではなく生成を扱う。
- CLI コマンド、入出力先、ファイル探索、永続状態など、生成された Markdown をどこで使うかを調べたいとき。呼び出し側の処理を直接読む方がよい。
- 単なる文字列整形全般やパス表記の定義を調べたいとき。この対象で扱う正規化は構造化文書レンダリングに付随する本文整形に限られる。
