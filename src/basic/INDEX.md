# `acp.py`

## Summary
- AI コーディングエージェント呼び出しに渡す論理パラメータを定義する基本モデル。モデルクラス、Reasoning effort、ファイルアクセスモードの抽象値と、プロンプト・Structured Output schema・書き込み許可パスをまとめる不変データクラスを扱う。
- バックエンド固有の実モデル名や実ファイルアクセス指定へ変換する前段の、cmoc 内部で共有される呼び出し条件の型定義として読む入口になる。

## Read this when
- エージェント呼び出し時に指定できるモデル品質・推論量・ファイルアクセス権限の論理分類を確認したいとき。
- Codex CLI などの AI コーディングエージェントへ渡す呼び出しパラメータの構造、必須項目、任意項目を確認したいとき。
- Structured Output schema の有無や、read-only 制約から例外的に外す書き込み可能パスを呼び出し単位でどう表すか確認したいとき。
- バックエンド実装で受理可能な具体的モデル名や権限設定へ解決するため、cmoc 側の抽象入力値を参照したいとき。

## Do not read this when
- 具体的なバックエンド名、実モデル名、API 呼び出し形式、権限フラグへの変換処理を探しているとき。この対象は抽象パラメータの定義だけを扱う。
- 実際にエージェントプロセスを起動する処理、コマンド実行、標準入出力、サブプロセス制御を調べたいとき。
- プロンプト本文の生成規則やテンプレート内容を調べたいとき。この対象は生成済みプロンプトを保持するだけで、本文の組み立て責務は持たない。
- Structured Output schema 自体の内容や検証規則を調べたいとき。この対象は schema ファイルパスを保持するだけで、schema の定義内容は扱わない。

# `path_model.py`

## Summary
- cmoc で使うルートトークン付きパス表記と、実パスとの相互変換を扱う実装。絶対パス、`<cmoc-root>`・`<repo-root>`・`<run-root>`・`<work-root>` 付き相対パスを受け取り、各ルートの探索規則に基づいて現実の絶対パスへ解決する。
- ルートトークンの定義、ルートディレクトリ探索、ルートトークンなし相対パスの拒否、実パスを指定ルートトークン表記へ戻す処理を確認する入口になる。

## Read this when
- cmoc 内でパス文字列や `Path` を実際の絶対パスへ解決する挙動を確認・変更したいとき。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味、探索起点、`.git` ディレクトリまたは `.git` ファイルによる判定条件を確認したいとき。
- ルートトークン付き相対パスを許可し、ルートトークンなし相対パスを禁止する入力検証の扱いを確認したいとき。
- linked worktree や git common dir を含む状況で、main worktree と run 用 worktree の判定がどう行われるかを追いたいとき。
- 実パスを特定のルートトークン基準の表記へ変換する処理や、対象外パスの場合の失敗条件を確認したいとき。

## Do not read this when
- CLI サブコマンドの引数定義、画面出力、終了コードなど、利用者向けコマンド挙動だけを確認したいとき。
- ルートトークンの概念ではなく、個別機能がどの作業ディレクトリでファイルを作成・更新するかという業務ロジックを確認したいとき。
- oracle file と realization file の所有関係や編集責務など、仕様管理上の分類だけを確認したいとき。
- テスト構成、fixture、テストケース追加先を探しているとき。

# `struct_doc.py`

## Summary
- 階層構造を持つ自然言語文書を Markdown に変換するための小さなヘルパーを定義する実装。見出し付きの構造化文書、挿入可能なコードブロック、Markdown 文字列へのレンダリング、連続空行の圧縮、三重引用文字列由来の本文正規化を扱う。
- 主な責務は、文書ツリーの親子関係から見出し深度を自動的に決め、本文文字列またはコードブロックを適切な Markdown として出力すること。複雑な Markdown パーサや汎用ドキュメント変換ではなく、内部で構築した単純な構造化文書を出力用 Markdown に整えるための部品である。

## Read this when
- 階層化された説明文、レポート、プロンプト、仕様断片などを Markdown 見出し付きで組み立てる処理を確認・変更したいとき。
- 見出し深度の自動計算、構造化文書の子要素として許容される型、コードブロックの info string と本文の扱いを確認したいとき。
- Markdown レンダリング時の空行圧縮、三重引用文字列風に書いた本文の先頭末尾空行除去や共通インデント解除の挙動を確認したいとき。
- 構造化文書を受け取る側・作る側の実装で、文字列本文、コードブロック、子文書リストのどれを渡せるかを調べたいとき。

## Do not read this when
- 既存の Markdown 文書を解析して構造化データへ変換する処理を探しているとき。この対象は Markdown の読み取りではなく、構造化済みデータのレンダリングを扱う。
- CLI コマンド、ファイルシステム操作、永続状態、パスモデル、Git 操作など、文書レンダリング以外のアプリケーション挙動を調べたいとき。
- INDEX.md のルーティング規則そのものや正本仕様断片の編集方針を確認したいとき。この対象はルーティング文書の仕様ではなく、Markdown 文字列を生成する実装部品である。
- 単なる文字列の dedent や空行圧縮だけを他の文脈で使う場所を探しているとき。対象内には補助関数があるが、主責務は構造化文書全体の Markdown レンダリングである。
