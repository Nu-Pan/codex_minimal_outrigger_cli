# `apply`

## Summary
- `cmoc apply fork` の差分要約・所見列挙・所見反映の 3 系統を束ねる入口。ここでは個別ロジックではなく、どの用途でどの下位ファイルへ進むかを判断する。
- 要約生成、ファイル単位の所見列挙、所見を受けた実装修正で役割が分かれているため、目的に応じて下位ファイルを選ぶためのルーティング対象として使う。

## Read this when
- `cmoc apply fork` の処理で、差分要約・所見列挙・所見反映のどれをたどるべきかを確認したいとき。
- fork 配下の各ファイルがそれぞれどの出力契約や作業段階を担当するかを知りたいとき。
- まず入口の責務分担を把握してから、対応する下位ファイルを直接読むべきか判断したいとき。

## Do not read this when
- 個別ファイルの prompt 本文、出力 schema、実装詳細を確認したいときは、対応する下位ファイルを直接読む。
- 差分取得や fork 作成など、apply 全体の実行制御を調べたいときは、この配下ではなく上位の apply 側を読む。
- 共通の prompt 生成規約や別サブコマンドの仕様を探しているときは、この配下ではなく該当する別領域を読む。

# `basic.py`

## Read this when
- Agent 呼び出しの入力条件や既定値を決めたいとき。
- モデル類型・推論強度・ファイルアクセス方針の候補や意味づけを確認したいとき。
- Structured Output を使う呼び出しと使わない呼び出しの違い、または indexing preflight を省く条件を確認したいとき。

## Do not read this when
- モデル名への具体的な解決方法を知りたいときは、変換処理側を読む。
- ファイルアクセス規則そのものの詳細を確認したいときは、規則定義側を読む。
- Agent 呼び出しの実行手順や入出力の組み立てを追いたいときは、呼び出し処理側を読む。

# `review`

## Summary
- `cmoc review oracle` の所見生成・整理・妥当性判定をまとめて扱う入口。所見の出し方、重複の扱い、理由収集、採否の契約を確認したいときに読む。

## Read this when
- 新しい所見をどう列挙し、既知の関連所見とどう重複排除するかを確認したいとき。
- 所見リストの重複解消や統合の方針を確認したいとき。
- 所見が妥当である理由・妥当ではない理由を、既知理由と重複しない形で集める契約を確認したいとき。
- 所見を人間へ提示すべきかどうかの判定結果と、その出力契約を確認したいとき。

## Do not read this when
- `cmoc review oracle` 全体の実行順や CLI の入出力だけを追いたいときは、呼び出し元の実行処理を読む。
- oracle file 全般の品質基準や、何を問題所見とみなすかという原則だけを確認したいときは、各種 oracle standard 側を読む。
- 個別の所見本文やレビュー対象の具体的な内容だけを確認したいときは、この層ではなく対象 oracle file を読む。

# `tui`

## Summary
- `cmoc tui` の起動前処理と実行パラメータ解決をまとめる入口。完全プロンプトの生成・保存、起動用 `AgentCallParameter` の組み立て、そして解決結果を返す schema を確認したいときにここから進む。
- TUI の画面操作や対話制御ではなく、起動前に決まる固定値と、元プロンプトをどう解決・保存するかに関心があるときの案内になる。

## Read this when
- `cmoc tui` の起動パラメータ、保存される完全プロンプト、または起動時に固定されるモデル・推論強度・ファイルアクセス方針を変えたいとき。
- 元プロンプトをどう共通プロンプトへ束ねて、どのファイルへ保存してからエージェント呼び出しに渡すかを確認したいとき。
- `cmoc tui` の実行フローのうち、TUI 起動用の `AgentCallParameter` 生成と、実行パラメータ解決用 schema のどちらへ進むべきかを切り分けたいとき。

## Do not read this when
- `cmoc tui` の実行本体、エディタ選択、対話 UI の制御を追いたいだけのとき。
- 実行パラメータ解決とは無関係に、TUI の個別 UI 部品や他サブコマンドの実装を見たいとき。
- `AgentCallParameter` 型そのものや共通のモデル定義だけを確認したいとき。
