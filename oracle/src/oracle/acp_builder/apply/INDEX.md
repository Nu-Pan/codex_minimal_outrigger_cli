# `fork`

## Summary
- `cmoc apply fork` の差分要約・所見列挙・所見対応を扱う一連の構成要素が集まる領域で、変更要約の出力契約、所見リストの出力契約、所見を受けた実装側の入口を分けて読むための導線になる。
- 変更要約の構造や主要変更箇所の持ち方を確認するなら要約定義側、ファイル単位の問題点リスト形式を確認するなら所見列挙側、所見を受けて AI 呼び出し条件や注意文を組み立てる流れを追うなら対応入口側を読む。

## Read this when
- fork 適用後の差分を人間向けの変更カテゴリ別サマリーとして出力・検証したいときは要約定義側を読む。
- 実装調査で見つかった問題点を、根拠位置や修正方針つきの所見リストとして返したいときは所見列挙の出力定義側を読む。
- 所見を受けて AI 呼び出しパラメータや入力プロンプトを組み立て、作業時の注意や実行条件を確認したいときは所見対応の入口を読む。
- 変更カテゴリ、主要変更箇所、空でない差分要約の前提のような出力互換性を固定したいときは要約定義側を読む。
- 所見に含めるべき根拠情報や、仕様要求・観測結果・理由・修正方針の粒度を確認したいときは所見列挙の出力定義側を読む。

## Do not read this when
- 個別ファイルのパッチ内容そのもの、diff 生成手順、git 操作の仕様を確認したいときは、この領域ではなく差分生成側を読む。
- fork の作成や適用、分岐制御など、差分要約や所見出力より前後の実行フローを調べたいときは、この領域ではなく前後工程の実装を読む。
- ルーティング文書そのものの書き方や一般的な INDEX.md の作法を確認したいときは、この領域ではなくルーティング文書側を読む。
- 所見を JSON 以外の文章、ログ、CLI 表示としてどう見せるかを確認したいときは、この領域ではなく表示側を読む。
- ファイル探索やパス解決の一般ロジックだけを見たいときは、この領域ではなく path 解決の共通実装を読む。
