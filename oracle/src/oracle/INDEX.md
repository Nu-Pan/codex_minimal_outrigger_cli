# `acp_builder`

## Summary
- oracle ACP builder の共通部品をまとめる oracle src 群の入口で、各 builder 実装で共有される状態・ルール・結果表現の正本仕様断片を読むための案内。個別 builder の詳細ではなく、複数領域にまたがる共通概念や型の境界を確認したいときに使う。

## Read this when
- ACP builder 全体で共通して使う状態、ルール、結果表現の正本仕様断片を確認したいとき。
- ACP builder の複数領域にまたがる挙動を実装・テストへ反映する前に、共有概念の境界を確認したいとき。
- 下位要素のどれを読むべきか、共通部品の責務から絞り込みたいとき。

## Do not read this when
- 特定の builder の個別仕様だけを確認したいとき。
- realization code 側の実装詳細やテスト構成を確認したいとき。
- ACP builder と無関係な oracle src の仕様を探しているとき。

# `other`

## Summary
- oracle 側の共通ヘルパー群への入口。`CmocConfig` の定義、ルートパス解決とプレースホルダ相互変換、構造化 Markdown レンダリング、規範 `Standard` から文書を組み立てる処理をまとめて確認したいときに読む。
- 個別のコマンド実装や業務ロジックではなく、oracle 側で再利用される基礎データ構造・変換ロジック・表現規則を横断して扱う層として案内する。

## Read this when
- oracle 側で共通に使う設定モデルや変換モデルを追加・変更したい。
- `{{cmoc-root}}` / `{{repo-root}}` / `{{run-root}}` / `{{work-root}}` の解決や、実パスとの相互変換を確認したい。
- 構造化された仕様文を Markdown にするレンダリング規則を確認したい。
- `Standard` と `Requirement` の保持形式や、規範文書への変換方法を確認したい。

## Do not read this when
- `cmoc` の各サブコマンドの実行手順や CLI 挙動を追いたい。
- 設定の保存先実装や読み書きの手順そのものを探したい。
- 個別の UI、出力、テストの詳細挙動を見たい。
- oracle 側の他の機能や別カテゴリの実装を探したい。

# `prompt_builder`

## Summary
- `prompt_builder` の入口群をまとめる階層で、プレースホルダ型、完全プロンプト組み立て、規範文面の部品を読むための導線を置く。`oracle` と `realization` の責務境界、組み立て順、注入条件、ルーティング規則をまたいで確認したいときにここから進む。
- この階層を読む目的は、個別の標準本文や実装詳細に入る前に、どの入口がどの論点を扱うかを見分けることにある。`basic` は置換対象の共通表現、`complete_prompt` は全体の組み立て規則、`parts` は規範文面の共通部品、`prompt_` は組み立て調停の入口として読む。

## Read this when
- プレースホルダ展開の型の意味を確認したいとき。
- agent call 用のプロンプト全体の並び順や、どの標準断片をどの条件で含めるかを変えたいとき。
- `oracle` と `realization` の基本区分、各種標準、レビュー規範、ファイルアクセス規則、INDEX.md のルーティング規範を見直したいとき。
- 完全なプロンプトの構成順や、依存する補助標準が自動で含まれる条件を確認したいとき。

## Do not read this when
- プロンプト本文の生成手順や置換ロジックの詳細を知りたいだけなら、実装側を読む。
- プレースホルダを使わない処理や、別の設定値の表現を確認したいだけなら、この階層は読まない。
- 個別の標準文面そのものを確認したいだけなら、対応する標準定義を直接読む。
- ファイルアクセス規則やルーティング規則の本文だけを確認したいだけなら、それぞれの専用定義を直接読む。
- 実装コードやテストコードの修正方針だけを確認したいときは、realization 側の対象ファイルを直接読む。
