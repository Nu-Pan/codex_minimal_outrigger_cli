# `acp_builder`

## Summary
- `oracle/src/oracle/acp_builder` 配下の各 builder 領域への入口。`cmoc` の agent call 周辺で、用途別に分かれた正本仕様断片を案内する。
- 配下には、apply, review, session, tui の各処理と、共通部品 `common`、型定義 `basic.py`、INDEX.md エントリー生成契約 `indexing` がある。

## Read this when
- ACP builder のどの領域を読むべきか、まず入口を絞りたい。
- builder 間で共有する概念、型、出力契約、各サブコマンドの prompt や実行条件のどれを確認すべきか整理したい。
- 個別領域の仕様を読む前に、対象が apply / review / session / tui / 共通 / 型定義 / INDEX エントリー契約のどれかを判断したい。

## Do not read this when
- 個別の実装詳細やテストの中身だけを確認したい。
- ACP builder 以外の oracle 仕様を探している。
- 既に読む対象の下位領域が特定できているので、この入口で再度絞り込みたくない。

# `other`

## Summary
- `cmoc` の共通モデルと文書化・パス解決の基礎をまとめた領域。設定定義、ルートパス表現、規範文書の構造化モデル、Markdown レンダリング補助を確認したいときの入口になる。

## Read this when
- 設定値の集約・既定値・永続化形式を確認したいとき。
- `<cmoc-root>` などのルート表現、相対パス禁止の扱い、実パスとの相互変換を確認したいとき。
- 規範文書を保持するデータ構造や、Markdown 生成のための文書レンダリング補助を確認したいとき。

## Do not read this when
- CLI の個別サブコマンドや実行フローを知りたいだけなら、対応するコマンド側のルーティングを読む。
- 設定の読み書き実装やパス解決の詳細な処理順だけが目的なら、対応する実装ファイルを直接読む。
- 規範本文そのものや利用者向け文書内容を探しているだけなら、この共通基盤ではなく該当する文書側を読む。

# `prompt_builder`

## Summary
- `basic.py` は、プレースホルダ名と置換先の対応を表す最小の型 alias を定義する入口。プレースホルダ置換のデータ構造の意味を確認したいときに読む。
- `complete_prompt.py` は、役割・概要・ゴール・アクセス制限・補助プロンプトを統合して完全な agent prompt を組み立てる入口。標準プロンプトの注入順や依存補正を確認したいときに読む。
- `parts` は、prompt_builder の共通部品群を束ねる入口。oracle/realization の基本概念、各種 standard、レビュー規範、アクセス規則、ルーティング規則の責務分担を見分けたいときに読む。
- `prompt_.py` は、完全な agent prompt を高水準に組み立てる実装の入口。静的・動的プロンプトの分離、標準注入の波及、placeholder 統合のタイミングを確認・変更したいときに読む。

## Read this when
- プレースホルダ置換用の mapping の型を確認したいとき。
- agent call 用の完全なプロンプトがどの順序で構築されるかを確認したいとき。
- プロンプト部品の責務分担を見分けたいとき。
- 生成される StructDoc の並びや、プレースホルダーマップの統合タイミングを確認したいとき。

## Do not read this when
- プレースホルダを実際に展開・置換する処理の流れを知りたいとき。この対象は型 alias だけを定義しており、変換ロジックは含まない。
- 個別の標準プロンプト本文や各パーツの文面を確認したいだけなら、それぞれのパーツ定義を読む。
- 個別の標準本文やレビュー規範そのものの詳細を確認したいときは、各部品の本文を読む。
- PlaceholderMap や StructDoc のデータ構造そのものを確認したいときは、それらの定義元を読む。
