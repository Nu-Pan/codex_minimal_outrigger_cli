# `acp_builder`

## Summary
- `cmoc` の ACP builder 群をまとめる正本仕様の入口。apply, review, session, tui などの各生成処理と、それらで共有する基本型・共通概念・INDEX エントリー契約を読むためのルーティング先。

## Read this when
- ACP builder 配下のどの領域を読むべきかまだ特定できず、まず共通の正本仕様断片から入口を絞りたい。
- `cmoc` の agent call 生成に関わる各サブ機能の仕様を、個別実装に入る前に確認したい。
- apply, review, session, tui のいずれかが扱う入力条件、出力契約、共通型の境界を確認したい。
- INDEX.md エントリー生成の出力契約や、共通型の定義を基準にしたい。

## Do not read this when
- 特定のサブ機能の個別仕様だけを見たいときは、より下位の対象へ直接進む。
- 実装詳細やテストの挙動を確認したいときは、ここではなく対応する realization code を読む。
- ACP builder 以外の正本仕様や別サブコマンドの仕様を探している。

# `other`

## Summary
- `cmoc` の設定正本、ルートパス解決、規範定義の構造化モデル、Markdown 生成ヘルパーをまとめた基盤層。ここは個別サブコマンドの挙動ではなく、設定・パス・規範文書・構造化文書の共通表現と変換規則を確認するときの入口になる。
- 実装差を避けたいのは、設定の永続化形、ルート判定条件、規範オブジェクトの構成、Markdown への整形ルールであり、これらの正本仕様断片を読むために進む。

## Read this when
- `cmoc` のリポジトリ設定、既定値、永続化形式、列挙値の保存形式を確認したい。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の解決条件や、プレースホルダ付きパスの解決・逆変換を確認したい。
- 規範文書を構造化して表現するデータモデルや、要求ラベル付きの規範定義の持ち方を確認したい。
- 階層文書やコードブロックを Markdown に整形する規則、空行圧縮や triple quoted string の正規化を確認したい。

## Do not read this when
- 個別サブコマンドの入出力や実行フローだけを追いたいなら、各サブコマンド側を読む。
- 設定値やルート解決の利用箇所を追いたいだけなら、ここではなく参照先の実装側を読む。
- 規範本文そのものや、生成済み Markdown の内容を知りたいだけなら、ここではなく該当する正本文書を読む。
- INDEX.md の方針や oracle / realization の管理規則だけを確認したいなら、別のルーティング文書を読む。

# `prompt_builder`

## Summary
- agent call 用のプロンプトを、役割・目的・制約・各種標準文面・補助入力を組み合わせて構築する領域への入口。ここでは、標準文面の選択条件と、最終プロンプトへ統合される順序を見分ける。
- この配下には、プロンプトを組み立てる高水準の実装と、その構成要素となる文面部品群がある。どの標準を注入するか、静的要素と動的要素をどう分けるか、プレースホルダをいつ統合するかを確認するために読む。

## Read this when
- 完全な agent call プロンプトの構築順を確認・変更したいとき。
- oracle / realization / review / index entry など、どの標準文面を注入するかの条件関係を確認したいとき。
- 静的プロンプト、動的プロンプト、プレースホルダ定義、ファイルアクセス制限の統合位置を調べたいとき。
- 個別の標準文面そのものではなく、複数の文面部品をどう束ねるかを見たいとき。

## Do not read this when
- 個別の標準文面やルーティング規則の本文だけを確認したいときは、該当する部品定義を直接読む。
- プレースホルダや構造化ドキュメントの型そのものを確認したいときは、その定義元を読む。
- agent call への渡し先ではなく、呼び出し側の実行経路を追いたいときは、この領域ではなく上位の実装を読む。
