# `basic.py`

## Summary
- プレースホルダ名を実パスや文字列へ対応付ける型定義を置く。プロンプト組み立てで、置換対象の名前と置換先を共通の表現で扱いたいときに読む。

## Read this when
- プレースホルダ展開に使う型の意味を確認したいとき。
- 文字列と `Path` を混在させる置換対象の表現を統一したいとき。

## Do not read this when
- プロンプト本文の生成手順や置換ロジックの詳細を知りたいときは、実装側を読む。
- プレースホルダを使わない処理や、別の設定値の表現を確認したいだけのとき。

# `complete_prompt.py`

## Summary
- cmoc 用の完全なエージェントプロンプトを、役割・依頼概要・完了条件・各種標準・ファイルアクセス規則・ルーティング規則・補助プロンプトから組み立てるビルダー。プロンプト部品の順序、依存する標準の自動有効化、プレースホルダ定義の統合を担う。

## Read this when
- エージェントに渡すプロンプト全体の構成や、標準プロンプト部品の有効化条件を変更・確認するとき。
- 新しいプロンプト部品、補助静的・動的プロンプト、プレースホルダ定義の注入方法を扱うとき。

## Do not read this when
- 個別のプロンプト部品の文面や責務だけを確認したいときは、対応する parts モジュールを直接読む。
- プレースホルダの具体的な定義や入力型だけを確認したいときは、basic モジュールや各ビルダーを直接読む。

# `parts`

## Summary
- cmoc のプロンプトを構成する部品群。oracle・realization の基本定義、各種標準規範、ファイルアクセス規則、ルーティング規則などを個別に構築し、上位の prompt builder から利用される入口。

## Read this when
- oracle と realization の定義・責務境界をプロンプトへ組み込む処理を確認するとき
- oracle standard、realization standard、レビュー基準、INDEX.md 規範などの生成内容を変更・調査するとき
- ファイルアクセス制限や INDEX.md による文書ルーティング規則の生成処理を確認するとき

## Do not read this when
- 個別の oracle file や realization file の具体的な仕様・実装を調査するとき
- プロンプト全体の組み立てや StructDoc などの共通基盤を直接確認するとき
- Codex CLI の実行環境や sandbox 設定そのものを確認するとき
