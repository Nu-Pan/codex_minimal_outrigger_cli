# `complete_prompt.py`

## Summary
- agent call 用の完全なプロンプトを組み立てる入口で、静的・動的プロンプトの順序や依存関係、必要な標準断片の注入を決める。
- このファイルを読むのは、プロンプト構成の追加・順序変更・注入条件変更・プレースホルダ処理の変更を確認したいとき。
- 個別の標準断片の本文や各定義の詳細を追うより、まず全体の組み立て規則を確認したいときに読む。

## Read this when
- agent call に渡すプロンプトの組み立て順や、どの標準断片をどの条件で含めるかを変えたい。
- 静的プロンプトと動的プロンプトの分離方針、プレースホルダ定義の集約方法、フラグ間の依存関係を確認したい。
- プロンプト全体の構成だけを先に把握して、必要なら各標準断片の本文へ進みたい。

## Do not read this when
- 個別の標準文面そのものを確認したいなら、対応する標準定義側を直接読む。
- ファイルアクセス制御やルーティング規則の本文だけを確認したいなら、それぞれの専用定義を直接読む。
- 単なる呼び出し側の引数値や、組み立て済みプロンプトの実例だけを見たい場合は、この組み立て関数ではなく生成元や利用箇所を読む。

# `parts`

## Summary
- `oracle` と `realization` に関する規範文面を組み立てる部品群を置く場所。基本概念、oracle 側の標準、realization 側の標準、レビュー規範、ファイルアクセス規則、INDEX.md エントリー規範、ルーティング規則を分けて扱い、必要な論点だけを読む入口になる。
- この階層は、個別機能の仕様本文ではなく、どの規範文面を使うべきかを選ぶための共通基盤をまとめる。`oracle` と `realization` の責務境界を確認したいときや、レビュー・アクセス・ルーティングの基準を揃えたいときにここから進む。

## Read this when
- `oracle` と `realization` の基本区分や責務境界を確認したいとき。
- oracle 側に書くべき規範、realization 側に求める実装規範、レビュー所見の境界、ファイルアクセス規則、INDEX.md のルーティング規範を見直したいとき。
- どの規範文面を組み立てるべきかを決める前に、対象テーマに対応する共通部品を探したいとき。

## Do not read this when
- 特定の CLI 機能、状態ファイル、出力形式、テスト対象などの個別仕様を知りたいときは、ここではなく該当する具体の oracle 側本文を読む。
- 実装コードやテストコードの修正方針だけを確認したいときは、realization 側の対象ファイルを直接読む。
- INDEX.md エントリーの書き方そのものや、対象本文を読むべきかどうかの判断基準だけを確認したいときは、ルーティング規範の部品を読む。

# `prompt_.py`

## Summary
- `build_complete_prompt` の組み立て方を決める入口。静的・動的プロンプトの並び順、依存する標準群の注入条件、プレースホルダー統合の方針を確認したいときに読む。
- 個別の標準本文そのものではなく、どの標準をどの順で含めるか、どの条件で依存フラグが補完されるかを把握するための調停層。

## Read this when
- 完全なプロンプトの構成順や、どの補助標準が自動で含まれるかを確認したいとき。
- role, summary, goal, ファイルアクセス制約, 補助プロンプト, プレースホルダー定義をまとめて扱う入口を探しているとき。
- プロンプト断片を追加・削除する前に、既存の注入順と依存関係を確認したいとき。

## Do not read this when
- 各標準の本文内容そのものを読みたいときは、対応する個別の標準ファイルを直接読む。
- ファイルアクセス規則やルーティング規則の詳細だけを確認したいときは、対応する専用の標準ファイルを読む。
- プロンプトの組み立てロジックではなく、個別の StructDoc の実装や下位ユーティリティの内部を追いたいとき。
