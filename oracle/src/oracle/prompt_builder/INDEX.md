## Summary
- プレースホルダ名を実パスや文字列へ対応付ける型定義を置く。プロンプト組み立てで、置換対象の名前と置換先を共通の表現で扱いたいときに読む。

## Read this when

## Do not read this when
- プロンプト本文の生成手順や置換ロジックの詳細を知りたいときは、実装側を読む。
- プレースホルダを使わない処理や、別の設定値の表現を確認したいだけのとき。

# `complete_prompt.py`

## Summary
- cmoc 用の完全なエージェントプロンプトを、役割・依頼概要・完了条件・各種標準・ファイルアクセス規則・ルーティング規則・補助プロンプトから組み立てるビルダー。プロンプト部品の順序、依存する標準の自動有効化、プレースホルダ定義の統合を担う。

## Read this when
- エージェントに渡すプロンプト全体の構成や、標準プロンプト部品の有効化条件を変更・確認するとき。
- 新しいプロンプト部品、補助静的・動的プロンプト、プレースホルダ定義の注入方法を扱うとき。

## Do not read this when
- 個別のプロンプト部品の文面や責務だけを確認したいときは、対応する parts モジュールを直接読む。
- プレースホルダの具体的な定義や入力型だけを確認したいときは、basic モジュールや各ビルダーを直接読む。

# `parts`

## Summary
- cmoc のプロンプト生成部品群。oracle／realization の基本定義・各種標準規範・ファイルアクセス規則・INDEX.md ルーティング規則を構造化文書として構築する実装を含む。個別のプロンプト規範やルーティング挙動を調査する際の入口。

## Read this when
- oracle と realization の定義・責務境界・配置規則を確認するとき
- oracle standard、realization standard、レビュー基準、INDEX.md エントリー規範を変更・確認するとき
- ファイルアクセス制約や INDEX.md による文書ルーティングのプロンプト生成を調査するとき
- 対象となるプロンプト部品の具体的な生成内容を確認するとき

## Do not read this when
- 実際の oracle file や realization file の個別仕様・実装を調査するとき
- cmoc の CLI 処理やプロンプト全体の組み立て、StructDoc など共通基盤だけを調査するとき
- INDEX.md の実際のエントリー内容を確認・編集するとき
