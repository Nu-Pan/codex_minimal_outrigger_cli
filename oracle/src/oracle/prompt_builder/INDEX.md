# `complete_prompt.py`

## Summary

- 各種 standard をフラグで注入する入口であり、依存する標準の組み合わせ調整、注入順序、プレースホルダの合成方法を確認したいときに読む。

## Read this when
- 完全な agent 用プロンプトをどう構成しているか知りたいとき。
- oracle / realization の基本情報や各 standard を、どの順序と依存関係でプロンプトに入れるか確認したいとき。
- 静的プロンプトと動的プロンプトの分離方針、またはプレースホルダ定義のまとめ方を変えたいとき。

## Do not read this when
- 個別 standard の本文そのものを確認したいときは、それぞれの build 関数の本文を読む。

- 呼び出し側のコマンド処理や agent の業務ロジックを追いたいときは、このファイルではなく利用側の実装を読む。

# `parts`

## Summary
- oracle/realization の各標準文面や `INDEX.md` ルーティング文を組み立てる部品群への入口。個別の標準本文を扱う前に、どの規範文面を読むべきか、また各部品がどの責務を持つかを見分けたいときに使う。
- この配下には、`oracle` の記述方針、`realization` の実装方針、`INDEX.md` の読む先の選び方、レビュー所見の判定、ファイルアクセス規則を担う部品があるため、正本仕様そのものではなく、それぞれの案内文を調整したいときに読む。

## Read this when

- `INDEX.md` で次に読む本文を案内する文面を調整したいとき。
- oracle file と realization file の境界、またはレビュー所見に含めるべき差分の境界を、共通の規範として確認したいとき。

## Do not read this when
- 特定の機能仕様や CLI 挙動など、個別の正本仕様そのものを確認したいとき。
- この配下の案内文ではなく、実際の実装やテストの修正方針を調べたいとき。
- `oracle` と `realization` の基本定義だけを確認したいときは、より基礎的な説明部品を読む方が直接的なとき。
