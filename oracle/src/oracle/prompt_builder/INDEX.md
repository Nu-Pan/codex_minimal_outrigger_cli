# `basic.py`

## Summary
- プロンプト生成などで使うプレースホルダ名と置換先文字列の対応表を表す型 alias を定義する、ごく小さな基本型定義モジュール。
- プレースホルダの key は `repo-root` のような名前、value は文字列または Path として扱う、という境界だけを示す入口。

## Read this when
- プレースホルダ置換用の mapping の型を確認したいとき。
- プロンプト構築処理で、プレースホルダ名から実際のパスや文字列へ置換するデータ構造を受け渡しする箇所を読む前に、基本型の意味を確認したいとき。
- 置換先 value に文字列だけでなく Path を許容するかどうかを確認したいとき。

## Do not read this when
- プレースホルダを実際に展開・置換する処理の流れを知りたいとき。この対象は型 alias だけを定義しており、変換ロジックは含まない。
- cmoc のパス概念そのもの、または `<cmoc-root>` や `<work-root>` などの意味を調べたいとき。パスモデルの定義を直接読む方が適切。
- プロンプト全体の組み立て順、出力文面、テンプレート内容を確認したいとき。

# `complete_prompt.py`

## Summary
- agent call に渡す完全なプロンプト列を構築する処理を定義する。役割・概要・完了条件・ファイルアクセス制限・ルーティング規則・任意の静的/動的プロンプトを、キャッシュ効率を意識した固定順序で組み立てる。
- oracle/realization/review/index entry などの標準注入フラグ間の依存を補正し、必要な前提プロンプトが欠けないようにしたうえで、プレースホルダ定義を末尾に追加する。

## Read this when
- agent call に渡す最終プロンプト全体の構築順序、静的プロンプトと動的プロンプトの配置、またはプロンプトキャッシュを意識した並びを確認・変更したいとき。
- oracle standard、realization standard、review standard、index entry standard などの注入フラグが、どの前提プロンプトを自動的に有効化するかを確認・変更したいとき。
- 追加プロンプトや追加プレースホルダ定義が、完全なプロンプトへどのように統合されるかを確認したいとき。
- ファイルアクセス規則やルーティング規則が、agent call 用プロンプトへ必ず含まれる経路を確認したいとき。

## Do not read this when
- 個別の標準文書パーツそのものの本文や構造を確認・変更したいだけのときは、各標準パーツの定義へ直接進む。
- ファイルアクセス制限の内容やルーティング規則の本文を確認したいだけのときは、それぞれの構築処理へ直接進む。
- プレースホルダ型の定義やファイルアクセスプロファイルの仕様を確認したいだけのときは、それらの型や仕様を定義する対象へ直接進む。

# `parts`

## Summary
- oracle file、realization file、ルーティング規則、レビュー規範、ファイルアクセス規則など、プロンプト本文へ差し込む個別の規範文書部品をまとめる領域。
- 各部品は、正本仕様断片や実装規律を StructDoc などのプロンプト用文書へ組み立てる責務を持ち、プロンプト全体ではなく特定規範の本文生成を扱う。
- 特定の規範文書部品がどの判断境界を定義しているか、またその部品を読むべき作業条件を選ぶ入口になる。

## Read this when
- プロンプトに埋め込む規範文書部品のうち、どの対象が oracle file、realization file、レビュー、ルーティング、ファイルアクセスのどの規範を生成しているかを選びたいとき。
- oracle file と realization file の基本概念、oracle 側の規範、realization 側の規範、レビュー所見の判定規範、INDEX.md エントリー規範のいずれかを確認する入口を探しているとき。
- プロンプト部品単位で生成本文を変更・確認する必要があり、プロンプト全体の組み立てではなく個別規範の責務境界から読む先を絞りたいとき。
- ファイルアクセスプロファイルをプロンプト文書へ変換する処理や、ルーティング規則をプロンプト化する処理など、特定の規則部品の生成責務を確認したいとき。

## Do not read this when
- プロンプト全体の組み立て順、複数部品の統合、placeholder 展開の全体制御を調べたいとき。
- 個別の CLI 機能、状態ファイル、パス解決、テスト方針など、プロンプト用規範部品ではないプロダクト仕様を確認したいとき。
- oracle file や realization file の実体そのものを編集・確認したいとき。
- ルーティング文書の既存エントリー内容だけを確認したいとき、または対象本文を読まずに機械的なファイル一覧やハッシュを確認したいだけのとき。

# `prompt_.py`

## Summary
- agent call に渡す完全なプロンプトを、標準プロンプト断片、任意の静的・動的プロンプト、ファイルアクセス制約、ルーティング規則、役割・概要・ゴールから組み立てる処理を担う。
- oracle・realization・review・INDEX エントリー関連の標準を注入するフラグ間の依存関係を補正し、プレースホルダー定義を集約して末尾に追加する。

## Read this when
- agent call 用の完全なプロンプト構築順序、静的プロンプトと動的プロンプトの配置、またはプロンプトキャッシュを意識した構成を確認・変更したいとき。
- 標準プロンプト断片を注入する boolean フラグの相互依存や、どの標準を有効化すると別の標準も自動的に含まれるかを確認したいとき。
- 追加プロンプトやプレースホルダー定義が、完成プロンプトへどの順序で反映されるかを確認したいとき。

## Do not read this when
- 個々の標準プロンプト断片の本文や構造を確認したいだけの場合は、それぞれの標準断片を構築する対象を直接読む。
- ファイルアクセス制約やルーティング規則そのものの内容を確認したいだけの場合は、それらを構築する対象を直接読む。
- プレースホルダー型や構造化ドキュメント型の定義だけを確認したい場合は、その型を定義する対象を直接読む。
