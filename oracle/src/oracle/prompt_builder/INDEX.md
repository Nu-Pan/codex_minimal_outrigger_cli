# `complete_prompt.py`

## Summary
- agent call に渡す完全なプロンプトの組み立て順序と、静的・動的プロンプトの配置方針をまとめる入口。
- 各種標準文の注入条件とフラグ依存の補正を扱うため、プロンプト全体の構成を変える変更で読む。

## Read this when
- 完全なプロンプトの構成順、先頭の固定案内、静的・動的要素の分離方針を確認したいとき。
- 標準文の有効化条件、フラグの依存関係、追加プロンプトやプレースホルダの扱いを変えたいとき。
- 複数の標準文をどう同時注入するか、どの順で積むかを追いたいとき。

## Do not read this when
- 個別の標準文そのものの内容を変えたいだけなら、対応する各 part 側を直接読む。
- 個別の StructDoc / StructBlock のデータ構造や表示形式だけを確認したいなら、ここではなくそれらの定義側を読む。
- プロンプト全体ではなく、単一の補助文や単一のルーティング規則だけを確認したいときは、この集約関数より該当 helper を先に読む。

# `parts`

## Summary
- `oracle` と `realization` の基本定義や、oracle 側・実装側・補助規範のどれを読むべきかを切り分けるための案内群。各文書は、レビュー基準、標準文、アクセス規則、ルーティング規則のように役割が分かれているときに、目的に合う本文へ進む入口として読む。
- `oracle` と `realization` の区分や、oracle standard / realization standard / review standard などの個別規範の責務境界を確認したいときに読む。対象本文を直接読む前に、どの規範が作業に関係するかを絞り込むための位置づけを持つ。

## Read this when
- oracle file と realization file の基本的な境界を確認したいとき。
- oracle 標準、realization 標準、レビュー基準、アクセス規則のどれを先に読むべきか迷うとき。
- INDEX.md で作業対象を絞るために、下位の案内文を確認したいとき。

## Do not read this when
- 個別の oracle 仕様断片や実装本文を直接確認したいとき。
- ファイルアクセスの実行時判定や具体的な実装ロジックだけを追いたいとき。
- レビュー所見の判定や実装整理の方針を、個別の標準本文で直接確認したほうが早いとき。
