# `basic.py`

## Summary
- プロンプト生成などで使うプレースホルダ名と置換先文字列の対応表を表す型 alias を定義する、ごく小さな基本型定義モジュール。
- プレースホルダの key は `repo-root` のような名前、value は文字列または Path として扱う、という境界だけを示す入口。

## Read this when
- プレースホルダ置換用の mapping の型を確認したいとき。
- プロンプト構築処理で、プレースホルダ名から実際のパスや文字列へ置換するデータ構造を受け渡しする箇所を読む前に、基本型の意味を確認したいとき。
- 置換先 value に文字列だけでなく Path を許容するかどうかを確認したいとき。

## Do not read this when
- プレースホルダを実際に展開・置換する処理の流れを知りたいとき。この対象は型 alias だけを定義しており、変換ロジックは含まない。
- cmoc のパス概念そのもの、または `<cmoc-root>` や `<work-root>` などの意味を調べたいとき。パスモデルの定義を直接読む方が適切。
- プロンプト全体の組み立て順、出力文面、テンプレート内容を確認したいとき。

# `complete_prompt.py`

## Summary
- agent call に渡す完全なプロンプト列を構築する入口。役割・概要・完了条件・ファイルアクセス制限・補助プロンプトを受け取り、静的プロンプト、動的プロンプト、プレースホルダ定義を一定順序で組み立てる責務を持つ。
- oracle、realization、review、index entry などの標準プロンプト注入フラグ間の依存関係を補正し、必要な標準パーツが欠けないようにしてから各プロンプトパーツを連結する。

## Read this when
- agent に渡す最終的なプロンプト全体の構築順序、静的プロンプトと動的プロンプトの分離、またはプレースホルダ定義の集約方法を確認したいとき。
- 標準プロンプト注入フラグの組み合わせにより、どの基礎情報や標準情報が追加で有効化されるかを確認したいとき。
- ファイルアクセス規則、ルーティング規則、任意の補助プロンプト、role、summary、goal が最終プロンプトへどの順序で入るかを確認したいとき。

## Do not read this when
- 個別の標準プロンプト本文やルール本文そのものを確認したいとき。この対象はそれらを組み立てる側であり、各パーツの内容は該当するパーツ定義を読む。
- プレースホルダ表現の基本型や構造化ドキュメントの表現を確認したいとき。この対象はそれらの型を利用するだけで、定義元ではない。
- ファイルアクセスモードの列挙値や意味を確認したいとき。この対象は制限設定を受け取ってパーツ構築へ渡すだけで、モード自体の定義元ではない。

# `parts`

## Summary
- AI エージェントへ渡す規範・ルール系プロンプトの本文断片を構築する部品群を置く領域。oracle file と realization file の基本概念、oracle 側・realization 側の標準、レビュー所見の判定基準、ルーティング規則、ファイルアクセス規則、INDEX.md エントリー規範など、cmoc の作業プロンプトで再利用される標準文書の生成責務を束ねる。
- 各部品は、対象規範を自然言語の構造化文書として組み立てる入口であり、プロンプト全体の流れそのものよりも、個別の判断基準や禁止事項を確認するための下位要素へ進む起点になる。

## Read this when
- AI エージェント向けプロンプトに含める標準・規則・レビュー基準の本文断片が、どの責務単位に分かれているか確認したいとき。
- oracle file、realization file、ルーティング、ファイルアクセス、レビュー所見、INDEX.md エントリーなど、複数の規範系プロンプト部品のうち読むべきものを選びたいとき。
- prompt builder が構築する標準文書や規則文書について、個別部品へ進む前に全体の分類と入口を把握したいとき。
- AI の作業規範やレビュー規範を構成する文章を変更・検証するため、どの部品が該当するか切り分けたいとき。

## Do not read this when
- 特定の標準や規則の本文内容をすでに特定できており、その個別部品へ直接進めばよいとき。
- プロンプト部品を結合する上位の組み立て順、placeholder 展開、StructDoc の汎用変換処理など、部品群の外側にある実装詳細を調べたいとき。
- 個別 CLI コマンド、状態ファイル、パス解決、テスト方針など、標準・規則プロンプト本文ではなくプロダクト仕様そのものを確認したいとき。
- realization implementation や realization test の現在構造を修正するために、実装コードや既存テストを直接読みたいとき。

# `prompt_.py`

## Summary
- agent call に渡す完全なプロンプトを、静的プロンプト、動的プロンプト、プレースホルダー定義の順で組み立てる責務を持つ。
- oracle、realization、review、apply、index entry などの標準プロンプト注入フラグを受け取り、必要な依存標準が欠けないように関連フラグを補正してから各パーツを追加する。
- 任意の追加静的・動的プロンプトと追加プレースホルダー定義を受け入れ、ファイルアクセス規則とルーティング規則を含む共通プロンプト構成の入口になる。

## Read this when
- agent に渡す最終的なプロンプト全体の構築順、構成要素、返却形式を確認したいとき。
- 標準プロンプト注入フラグ同士の依存関係や、どのフラグがどの標準プロンプトを追加するかを確認したいとき。
- 静的プロンプト、動的プロンプト、プレースホルダー定義、補助プロンプトがどの順序で結合されるかを調べたいとき。
- ファイルアクセス規則やルーティング規則が完全プロンプトへ常に含まれる経路を確認したいとき。

## Do not read this when
- 個々の標準プロンプトパーツの本文や規則内容を確認したいだけのときは、各プロンプトパーツの生成元を直接読む。
- プレースホルダー型や基本的なデータ構造の定義だけを確認したいときは、それらの定義元を読む。
- 生成されたプロンプトを実際にどのコマンドや agent 実行が利用するかを調べたいときは、呼び出し側の実装を読む。
