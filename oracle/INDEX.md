# `doc`

## Summary
- `app_spec` は、CLI 自動補完の起動判定と、補完プローブ中に通常実行の前処理や副作用を混ぜない境界を確認するための入口。補完時の出力を CLI ライブラリ向けに限定するかを判断するときに読む。

## Read this when
- シェル補完や CLI ライブラリの補完処理に関わる起動経路を実装・修正・テストするとき。
- 通常の CLI 実行前処理、サブコマンド未指定判定、作業ディレクトリ変更、状態検査、ログ作成、索引更新、独自エラー出力を補完前に実行してよいか判断するとき。
- 補完プローブ時に stdout/stderr へ余計な出力や副作用が混入していないか確認するとき。

## Do not read this when
- 通常実行時のサブコマンド仕様、状態ファイル仕様、ログ仕様、索引更新仕様そのものを調べたいだけのとき。
- 補完プローブではない通常の CLI エラー形式や出力仕様を確認したいとき。
- CLI 自動補完に関係しない oracle file と realization file の役割分担や品質基準を調べたいとき。

# `src`

## Summary
- oracle 側の共通基盤を束ねる入口で、設定・変換・文書化・規範文面の共通部品を横断して読むための案内。個別コマンドや個別 builder ではなく、再利用される基礎概念と変換規則の境界を確認したいときにここから進む。
- 下位の `acp_builder` は builder 間で共有される状態・ルール・結果表現の正本仕様断片を読むための入口として使う。`other` は共通ヘルパー、設定モデル、パス解決、Markdown レンダリング、規範文書組み立ての入口として使う。`prompt_builder` はプレースホルダ型、完全プロンプト組み立て、規範文面部品、ルーティング規則の入口として使う。

## Read this when
- oracle 側で共通に使う設定モデルや変換モデルを追加・変更したいとき。
- 複数の builder にまたがる状態、ルール、結果表現の正本仕様断片を確認したいとき。
- `{{cmoc-root}}` や `{{repo-root}}` などの解決、構造化 Markdown のレンダリング、`Standard` と `Requirement` の保持形式を確認したいとき。
- プレースホルダ展開、完全プロンプトの並び順、どの標準断片をどの条件で含めるか、ルーティング規則の入口を見分けたいとき。

## Do not read this when
- 特定の builder の個別仕様だけを確認したいとき。
- CLI の実行手順や realization 側の実装詳細、テスト構成を確認したいとき。
- 設定の保存先や読み書きの具体的な処理だけを追いたいとき。
- 個別の UI、出力、テストの詳細挙動だけを見たいとき。
- oracle 側の他の機能や別カテゴリの実装を探しているとき。
