# `doc`

## Summary
- `app_spec` は、このアプリに固有の正本仕様断片へ進む入口である。共通基盤ではなく、個別機能の外部挙動、起動条件、入出力、状態遷移、終了条件を確認したいときに読む。
- `branch_model.md` は、session 用 branch と run 用 branch / worktree の切り分け方を定める基礎仕様である。session fork の分岐元・merge 先、run ごとの隔離先、branch 名や worktree 名の対応を確認したいときに読む。
- `considered_alternative` は、採用しなかった設計案とその不採用理由を集めた領域である。現行仕様そのものではなく、apply orchestration、違反検査、ignore と permission profile の変換、自動注入、レビュー方式の再検討時に読む。
- `dev_rule` は、Python 実装・テストの共通記法と記述方針を確認する入口である。型ヒント、import、docstring、コメント、ログ、命名、参照方法など、実装差を避けたい書き方の判断に使う。

## Read this when
- このアプリの仕様、実装、テストを追加・修正するとき。
- session 作成時の分岐元や最終 merge 先を決める必要があるとき。
- cmoc の現行設計に対して、過去に不採用となった代替案を再検討しているとき。
- Python の実装またはテストを追加・修正し、型ヒント、import、docstring、コメント、ログメッセージ、非公開識別子の書き方を確認したいとき。

## Do not read this when
- 共通の CLI 基盤、ログ規約、エラー処理、索引更新、事前検証などの横断仕様だけを確認したいとき。
- git の一般的な branch 運用や worktree 機能そのものを学びたいだけで、cmoc 固有の対応関係は不要なとき。
- 採用済みの詳細仕様、CLI 入出力、状態ファイル仕様、テスト期待値、実装経路を確認したいとき。
- CLI の具体的なコマンド仕様、出力 schema、保存状態、エラー条件など、利用者に見える個別機能の正本仕様を確認したいとき。

# `src`

## Summary
- `oracle/src/oracle` 配下の正本仕様を、機能領域ごとに読むための入口。ACP builder、基盤設定、プロンプト組み立ての各領域へ進む前に、どの責務がどこにあるかを切り分けるためのルーティング先。
- 個別サブ機能へ入る前に、共通概念や境界条件を確認したいときに使う。実装詳細ではなく、正本として読むべき仕様断片の所在を絞る役割が中心。

## Read this when
- `oracle/src/oracle` 配下で、まずどの領域の正本仕様を読むべきかを切り分けたい。
- ACP builder、設定・パス・共通モデル、プロンプト構成のどれに関係するかを先に判断したい。
- 正本仕様の断片を読む前に、共通の基盤仕様と個別責務の境界を確認したい。

## Do not read this when
- 特定サブ機能の個別仕様だけを見たいなら、より下位の対象へ直接進む。
- 実装やテストの挙動を確認したいなら、対応する realization 側を読む。
- ACP builder 以外の領域や、別の正本仕様を探しているなら、この入口ではなく該当領域へ進む。
