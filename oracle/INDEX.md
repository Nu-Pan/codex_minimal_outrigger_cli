# `doc`

## Summary
- `oracle/doc` 配下の正本仕様断片をまとめて読むための入口。cmoc 全体の仕様をテーマ別に分けた下位文書へ進む起点として使う。
- ここでは全体像の把握と読む先の選別だけを担う。利用手順、サブコマンド、状態管理、ログ、エラー、補完、実行分離、設計判断の入口を探すときに読む。

## Read this when
- cmoc の仕様をどのテーマから読むべきか判断したいとき。
- サブコマンド全体の利用手順や共通規則を先に押さえたいとき。
- どの下位文書が目的に直結するかを見分けて、そこへ直接進みたいとき。

## Do not read this when
- 個別サブコマンドの入力や出力だけを確認したいときは、該当する下位文書を直接読む。
- 実装の細部やコード変更点だけを追いたいときは、この階層ではなく対象実装へ進む。
- `oracle` と `realization` の一般定義だけを確認したいときは、この階層ではなく上位の定義文書を読む。

# `src`

## Summary
- `oracle/src/oracle` 配下の仕様断片と生成入口を束ねるルーティング対象。共通定義と各サブ領域の責務分担を切り分けたいときの入口で、個別の正本本文や実装詳細をここで完結させない。
- 共通の呼び出しパラメータ型、設定、パス解決、標準文レンダリングの基盤を確認したいときは、この階層から対応する下位定義へ進む。
- `cmoc apply`、`cmoc indexing`、`cmoc review`、`cmoc session join`、`cmoc tui` 向けの agent 呼び出し正本を探すときは、この階層からサブコマンド別領域へ進む。

## Read this when
- oracle 側の共通型や基盤定義を探していて、まず読む対象を切り分けたいとき。
- 特定サブコマンドの prompt 生成や Structured Output の入口を探していて、共通定義と個別定義のどちらを読むべきか判断したいとき。
- 設定、パス解決、標準文レンダリングのいずれかを変更したくて、その責務の所在を確認したいとき。

## Do not read this when
- 個別サブコマンドの prompt 本文、出力契約、実行条件を確認したいときは、対応する下位領域を直接読む。
- 共通の設定定義、パス解決、標準文レンダリングの詳細だけを確認したいときは、対応する定義元を直接読む。
- agent 呼び出しの実行フロー全体や CLI 側の制御を追いたいときは、この階層ではなく実行側を読む。
