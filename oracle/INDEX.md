# `doc`

## Summary
- `oracle/doc` は cmoc の正本仕様断片をまとめる入口です。共通の実行前提、利用手順、サブコマンド群、設計判断、開発規則へ進む起点になります。
- `app_spec/` は補完・ログ・エラー・プロンプト・隔離・状態管理・利用手順など、cmoc の共通仕様と個別サブコマンド仕様の入口です。
- `branch_model.md` は session と run の境界、branch と worktree の責務分担を確認したいときに読む文書です。
- `considered_alternative/` は採用しなかった設計案とその不採用理由を確認したいときの入口です。
- `dev_rule/` は Python 実装、CLI 構成、テストの共通ルールを確認したいときの入口です。

## Read this when
- cmoc 全体の仕様のどこから読むべきかを判断したいとき。
- 共通前提から個別サブコマンドへ進む前に、関連する正本仕様断片を探したいとき。
- session/run の分離、設計判断の背景、開発時の共通ルールを確認したいとき。

## Do not read this when
- すでに対象サブコマンドや対象ルールが分かっていて、該当本文へ直接進めるとき。
- 個別の実装詳細や挙動の本体を知りたいだけで、ルーティング情報が不要なとき。
- 採用中の仕様ではなく、一般論だけを確認したいとき。

# `src`

## Summary
- `oracle/src/oracle` 配下の正本仕様断片を束ねる入口。ACP builder の共有基盤、cmoc の共通基盤、prompt 組み立て部品のどれを読むべきかを、扱う概念から切り分けるために使う。

## Read this when
- oracle src 全体の中で、まずどの共通基盤を読むべきかを絞り込みたいとき。
- ACP builder の共有状態、共通ルール、結果表現にまたがる仕様を確認したいとき。
- cmoc の設定、パス解決、構造化文書にまたがる仕様を確認したいとき。
- prompt 組み立ての共通部品に触れる前に、共有概念の責務境界を確認したいとき。

## Do not read this when
- 特定機能の個別仕様だけを確認したいときは、対応する下位エントリーを直接読む。
- realization code の実装詳細やテスト構成を確認したいときは、この配下ではなく実装側を読む。
- oracle src の共通基盤ではなく、別系統の正本仕様断片を探しているとき。
