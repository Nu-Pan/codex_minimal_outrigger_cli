# `doc`

## Summary
- cmoc 全体に関わる正本仕様断片群への入口。CLI 起動前処理、補完、ログ、エラー、サブコマンド入口、実行隔離、セッション状態、外部 provider 境界、利用手順のような横断仕様をまとめ、個別実装へ進む前に読むべき範囲を切り分ける。
- 個別サブコマンドの詳細よりも、どの仕様断片を先に読むべきかを判断するために使う。

## Read this when
- CLI 全体の起動経路や共通前処理を実装・修正するとき。
- 補完、ログ、エラー処理、実行隔離、セッション状態、外部 provider との責務分担などの境界を確認したいとき。
- どのサブコマンド仕様や周辺仕様へ進むべきかを判断したいとき。

## Do not read this when
- 個別サブコマンドの入出力や内部手順だけを確認したいときは、対応する個別仕様を直接読む。
- 単一の実装ファイルやテストファイルの細部だけを追いたいときは、より下位の realization file を読む。
- oracle file と realization file の一般原則だけを確認したいときは、この配下ではなくそれぞれの共通ルールを読む。

# `src`

## Summary
- `oracle/src/oracle` 配下の正本仕様を、機能領域ごとに読むための入口。ACP builder、基盤設定、プロンプト組み立ての各領域へ進む前に、どの責務がどこにあるかを切り分けるためのルーティング先。
- 個別サブ機能へ入る前に、共通概念や境界条件を確認したいときに使う。実装詳細ではなく、正本として読むべき仕様断片の所在を絞る役割が中心。

## Read this when
- `oracle/src/oracle` 配下で、まずどの領域の正本仕様を読むべきかを切り分けたい。
- ACP builder、設定・パス・共通モデル、プロンプト構成のどれに関係するかを先に判断したい。
- 正本仕様の断片を読む前に、共通の基盤仕様と個別責務の境界を確認したい。

## Do not read this when
- 特定サブ機能の個別仕様だけを見たいなら、より下位の対象へ直接進む。
- 実装やテストの挙動を確認したいなら、対応する realization 側を読む。
- ACP builder 以外の領域や、別の正本仕様を探しているなら、この入口ではなく該当領域へ進む。
