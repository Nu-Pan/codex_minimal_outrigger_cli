# `doc`

## Summary
- `app_spec` は cmoc 全体の仕様の入口で、個別の正本断片へ進む前に、どの文書を読むべきかを切り分けるための案内役である。
- `branch_model.md` は branch と worktree、session/run の関係を確認するときの入口で、`session fork` / `session join` / `apply` / `review` の責務境界を読むために使う。
- `considered_alternative` は採用しなかった設計案とその理由を集めた入口で、現行仕様そのものではなく代替案の背景を確認したいときに読む。
- `dev_rule` は開発時の環境前提、Python の記述規則、CLI の構成、pytest ベースの realization test の方針を確認するときの入口である。

## Read this when
- cmoc の全体仕様のどこを読むべきか迷っているとき。
- 複数の仕様断片にまたがる振る舞いの境界や優先関係を確認したいとき。
- branch と worktree、session/run の分離や命名の意味を把握したいとき。
- 採用しなかった設計案の背景や不採用理由を確認したいとき。
- 作業環境、Python の書き方、CLI の責務分担、pytest ベースのテスト方針を確認したいとき。

## Do not read this when
- すでに読むべき個別の正本断片が分かっているとき。
- 特定機能の詳細仕様、実装手順、状態遷移だけを確認したいとき。
- 現行の CLI 振る舞いそのものではなく、却下された案の背景が不要なとき。
- 一般的な git の説明だけが必要で、この文書の責務まで要らないとき。
- 仕様そのものではなく、単なる実装や生成物だけを探しているとき。

# `src`

## Summary
- oracle 側の共通基盤を束ねる入口で、設定・変換・文書化・規範文面の共通部品を横断して読むための案内。個別コマンドや個別 builder ではなく、再利用される基礎概念と変換規則の境界を確認したいときにここから進む。
- 下位の `acp_builder` は builder 間で共有される状態・ルール・結果表現の正本仕様断片を読むための入口として使う。`other` は共通ヘルパー、設定モデル、パス解決、Markdown レンダリング、規範文書組み立ての入口として使う。`prompt_builder` はプレースホルダ型、完全プロンプト組み立て、規範文面部品、ルーティング規則の入口として使う。

## Read this when
- oracle 側で共通に使う設定モデルや変換モデルを追加・変更したいとき。
- 複数の builder にまたがる状態、ルール、結果表現の正本仕様断片を確認したいとき。
- `{{cmoc-root}}` や `{{repo-root}}` などの解決、構造化 Markdown のレンダリング、`Standard` と `Requirement` の保持形式を確認したいとき。
- プレースホルダ展開、完全プロンプトの並び順、どの標準断片をどの条件で含めるか、ルーティング規則の入口を見分けたいとき。

## Do not read this when
- 特定の builder の個別仕様だけを確認したいとき。
- CLI の実行手順や realization 側の実装詳細、テスト構成を確認したいとき。
- 設定の保存先や読み書きの具体的な処理だけを追いたいとき。
- 個別の UI、出力、テストの詳細挙動だけを見たいとき。
- oracle 側の他の機能や別カテゴリの実装を探しているとき。
