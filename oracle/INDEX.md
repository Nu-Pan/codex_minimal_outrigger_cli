# `doc`

## Summary
- `app_spec/` は cmoc の共通前提と個別サブコマンド仕様の入口です。補完・ログ・エラー・プロンプト・隔離実行・状態管理の共通断片を先にそろえたいとき、または `apply` / `session` / `doctor` / `review` / `indexing` / `tui` の個別責務へ進みたいときに読む対象です。
- `branch_model.md` は session と run の境界、誰の作業領域か、branch と worktree の分離を確認したいときに読む対象です。サブコマンドの実行場所や merge の責務境界を決めるときに進みます。
- `considered_alternative/` は採用しなかった設計案とその理由を確認したいときに読む対象です。現在の仕様ではなく、別案を退けた背景や判断根拠を追いたいときに進みます。
- `dev_rule/` は Python 実装規約、CLI の配置方針、開発環境、pytest の使い方を確認したいときに読む対象です。実装やテストを作る前に、書き方や置き場所の約束をそろえたいときに進みます。
- `external_model_provider.md` は本文が空なので、この階層の routing 先としては使いません。実仕様が追記されたときだけ読む対象になります。

## Read this when
- cmoc の共通仕様と個別サブコマンド仕様のどちらへ進むべきか判断したいとき。
- branch / worktree / session / run の責務境界を確認したいとき。
- 採用しなかった設計案や、その不採用理由を確認したいとき。
- Python 実装規約、開発環境、テスト方針を確認したいとき。
- 外部 model provider の本文仕様が追加されたかを確認したいとき。

## Do not read this when
- 個別サブコマンドの入力条件や状態遷移をすでに知っているなら、`app_spec/` ではなく該当文書へ直接進みます。
- branch 名や worktree 名の具体的な生成規則ではなく、実装詳細だけを詰めたいときは `branch_model.md` は先送りでよいです。
- 現行仕様そのものを探していて、不採用案の背景が不要なときは `considered_alternative/` は読まなくてよいです。
- CLI 挙動や出力仕様ではなく、Python の書き方やテスト規約だけを知りたいときは `dev_rule/` に進みます。
- `external_model_provider.md` が空のままなら、この文書単体では判断しません。

# `src`

## Summary
- `oracle/src` 配下の共通仕様断片を読む入口。ACP builder の共有状態・ルール・結果表現、cmoc の設定/パス解決/文書表現、プロンプト生成の共通材料を先に確認したいときに使う。個別手順や実装詳細へはここから直接入らない。

## Read this when
- ACP builder 全体で共通して使う状態、ルール、結果表現の正本仕様断片を確認したいとき。
- ACP builder の複数領域にまたがる挙動を実装・テストへ反映する前に、共有概念の境界を確認したいとき。
- cmoc の永続設定モデルや、その既定値・保存方針を確認したいとき。
- ルートパスの解決や、実パスとの相互変換を確認したいとき。
- 規範文書を保持するデータ構造や、markdown への落とし込み方を確認したいとき。
- 階層付き markdown のレンダリングや、`cmoc_ref` と `cmoc_block` の検査、文字列やコードブロックの正規化を確認したいとき。
- プレースホルダ展開で使う型の意味を確認したいとき。
- 完全な agent 用プロンプトの構成や、静的な規約と可変要素のまとめ方を確認したいとき。
- oracle と realization の基本区分や、各種標準文面の責務境界を確認したいとき。

## Do not read this when
- 特定の builder の個別仕様だけを確認したいとき。
- realization code 側の実装詳細やテスト構成を確認したいとき。
- ACP builder と無関係な oracle src の仕様を探しているとき。
- `cmoc` の各サブコマンドの実行手順や CLI の入出力を追いたいとき。
- 設定読み書きの具体的な永続化処理や、サブコマンド固有の業務ロジックを見たいとき。
- 構造化文書やパス解決以外の別系統の共通処理を探したいとき。
- プロンプト本文の生成手順や置換ロジックの詳細だけを知りたいとき。
- 個別の標準本文そのものを確認したいとき。
- 実装コードやテストコードの修正方針だけを確認したいとき。
