# `coding_rule.md`

## Summary
- cmoc の Python 実装で守るべきコーディング規則を定める正本仕様断片。PEP 8、型ヒント、import、docstring、コメント言語、ログ言語、非公開識別子の扱いなど、realization code を書く際の基本的な作法への入口になる。
- 実装差を避けたいコード記述上の規則を扱い、機能仕様そのものや個別コマンドの入出力仕様ではなく、実装・テスト全体に共通する書き方の基準を示す。

## Read this when
- Python の実装またはテストを追加・修正し、型ヒント、import、docstring、コメント、ログメッセージ、非公開識別子の書き方を確認したいとき。
- コードレビューや実装修正で、コメントの粒度、日本語コメントと英語ログの使い分け、NOTE を付けるべき補足の扱いを判断したいとき。
- 新しい関数・クラス・モジュールを作る前に、命名、責務、入出力を明確に保ち、過剰な実装を避けるための共通規則を確認したいとき。
- 相対 import、循環参照、TYPE_CHECKING の使用可否など、モジュール間参照の書き方を判断したいとき。

## Do not read this when
- CLI の具体的なコマンド仕様、出力 schema、保存状態、エラー条件など、利用者に見える個別機能の正本仕様を確認したいとき。
- oracle file と realization file の役割、正本仕様断片の扱い、仕様と実装の関係など、リポジトリ全体の仕様管理原則を確認したいとき。
- ファイル分割、重複削減、依存追加、テスト肥大化抑制など、realization file の保守量や構成判断に関する一般原則を確認したいとき。
- 対象コードの具体的な実装場所や既存実装の現在の構造を探したいとき。

# `design_rule.md`

## Summary
- CLI 構成と共通処理の配置方針を定めるコード設計ルール。typer による CLI 実装、エントリーポイントとサブコマンド本体の責務分離、複数サブコマンドで使う機能の共通配置を判断するための正本仕様断片である。

## Read this when
- CLI 引数解釈、エントリーポイント、サブコマンド呼び出し構造を実装または変更する。
- 新しいサブコマンドを追加し、その本命処理をどこに置くか判断する。
- 複数のサブコマンドから使う定数、文字列処理、エラー処理などの共通機能を配置する。
- CLI 定義側に処理を直接書くべきか、サブコマンド実装へ委譲すべきかを確認する。

## Do not read this when
- 個別サブコマンドの外部仕様、入出力形式、状態更新条件を確認したいだけで、CLI 配置や共通機能配置を判断しない。
- path キーワードの意味、oracle file と realization file の関係、INDEX.md 生成ルールなど、コード設計以外の基本概念を確認したい。
- 既存実装の具体的な関数シグネチャ、内部ロジック、テスト期待値を調べたい。

# `development_environment.md`

## Summary
- cmoc の開発環境に関する正本仕様断片。想定する OS・エディタ・ワークスペース、ファイルエンコード、ファイル名・ディレクトリ名の命名規則、Python 仮想環境と pip の使い方、仮想環境の作成・インストール・依存追加手順を扱う。
- 開発作業や環境構築時に、どの Python インタプリタや pip を使うべきか、どの命名規則やエンコード方針に従うべきかを確認する入口となる。

## Read this when
- cmoc の開発環境を準備する、または想定環境が仕様上どう定められているか確認したいとき。
- Python 実行、pip 実行、仮想環境作成、パッケージインストール、依存関係追加の手順や禁止事項を確認したいとき。
- 新しいファイルやディレクトリを作る前に、命名規則やエンコード方針を確認したいとき。
- 開発用エディタやワークスペースの前提を確認したいとき。

## Do not read this when
- CLI の機能仕様、出力形式、サブコマンドの挙動を知りたいとき。
- 実装コードやテストコードの設計・責務分割・品質基準を確認したいとき。
- oracle file と realization file の関係、正本仕様断片としての扱い、INDEX.md エントリーの作成基準を確認したいとき。
- 特定の実装ファイル、テストファイル、または実行時のパスモデルの詳細を調べたいとき。

# `test_rule.md`

## Summary
- cmoc の自動テスト実装規約を定める oracle doc。pytest の使用、realization test の配置、検証対象を決定論的な制御ロジックに限定する方針、テスト環境を tmp_path 配下に閉じる方針、Codex CLI 呼び出し時の有料モデル禁止、ローカル SLM や Fake Codex CLI の使い分けを扱う。

## Read this when
- cmoc の realization test を追加・修正する際に、テストで何を検証対象にしてよいか判断したいとき。
- pytest、tmp_path、テスト用リポジトリ、Codex CLI 呼び出しを含むテスト環境の作り方を確認したいとき。
- Codex CLI や LLM を伴うテストで、有料クラウドモデル、ローカル SLM、Fake Codex CLI のどれを使うべきか判断したいとき。

## Do not read this when
- 実装コード本体の責務分割や公開面の増減など、テスト以外の realization implementation 規約を確認したいとき。
- Codex CLI や LLM の出力品質そのものを評価する方法を探しているとき。
- pytest を使わない外部テスト基盤や、本番実行時の Codex CLI バックエンド設定を設計したいとき。
