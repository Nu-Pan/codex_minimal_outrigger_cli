# `builder`

## Summary
- AI エージェント呼び出し用のパラメータ構築を担う領域であり、目次エントリー生成と TUI 実行前パラメータ解決の二系統を入口として持つ。prompt 構成、読み取り権限、標準文書参照、モデル種別、reasoning effort、Structured Output schema の接続関係を追うための場所である。

## Read this when
- `cmoc indexing` で、対象パスと対象内容からルーティング文書用エントリー生成の AI 呼び出し条件がどう作られるか確認したいとき。
- `cmoc tui` で、ユーザー入力プロンプトからファイルアクセスモード、標準文書の参照要否、モデル・推論強度、出力 schema がどう選ばれるか確認したいとき。
- AI 呼び出しに渡す role、summary、goal、補助入力、complete prompt、Structured Output schema 保存先の対応関係を、呼び出しパラメータ構築側から調べたいとき。
- 目次エントリー生成または TUI パラメータ解決の出力検証で要求される structured output の項目と制約を確認したいとき。

## Do not read this when
- CLI サブコマンド全体の実行制御、対象探索、既存目次の読み書き、TUI の画面操作や起動フローを調べたいとき。
- oracle、realization、review、apply review、index entry の各標準文書そのものの要求内容を理解したいとき。
- 共通の complete prompt 構築、Markdown レンダリング、パス解決、ファイルアクセス規則文、AgentCallParameter 型定義の内部実装だけを調べたいとき。
- 個別対象の実際の本文を読んでルーティング内容を評価したいだけのとき、または生成済みの INDEX.md 本文を確認したいだけのとき。

# `prompt_parts`

## Summary
- エージェント呼び出しに渡す構造化プロンプト部品を組み立てる実装群である。基本プロンプト、ファイルアクセス規則、ルーティング規則、oracle と realization の基本説明、各種標準文書、レビュー基準、案内文作成基準を扱う。
- 個別の規範文書を構造化文書へ変換する小さなビルダーと、それらを依存関係に応じて完全なプロンプト列へ統合する入口を含む。
- プロンプト本文として生成される規範や説明の内容を確認・変更するための対象であり、実際の CLI 実行、ファイル探索、パス解決、構造化文書型そのものの汎用実装へ進む前の入口になる。

## Read this when
- エージェントに渡す最終プロンプトの構成、各プロンプト部品の注入順序、または標準文書間の依存関係を確認・変更したいとき。
- ファイルアクセス規則、ルーティング規則、oracle と realization の基本概念説明、oracle 標準、realization 標準、レビュー標準、案内文作成標準として生成される文面を確認・変更したいとき。
- 特定の作業モードや標準注入フラグによって、どの構造化プロンプトが agent call に含まれるかを追いたいとき。
- oracle file や realization file をレビューする際の所見判定基準、または案内文を作る際の品質基準を、生成されるプロンプト本文側から確認したいとき。

## Do not read this when
- 構造化文書、標準、要求、ファイルアクセスモードなどの基礎データ型や変換 helper の汎用実装だけを調べたいとき。
- 実際の CLI コマンド、レビュー実行フロー、ファイル走査、入出力、保存先、パス解決など、生成されたプロンプトを使う側の挙動を調べたいとき。
- 個別の oracle file や realization file の正本仕様内容、実装内容、テスト内容を理解したいとき。
- 既に生成された案内文やレビュー結果の内容そのものを確認したいとき。
