# `oracle`

## Summary
- 正本仕様断片レビューに関する AI 呼び出しパラメータ構築実装と、それらが参照する Structured Output schema をまとめる領域。
- レビュー対象の仕様断片から新規所見を列挙し、所見の妥当性を擁護・反証し、人間提示の採否を判定し、複数所見を整理・統合する各段階への入口になる。
- 各実装は `cmoc review oracle` のレビュー用サブタスクに渡すロール、ゴール、補助プロンプト、標準文書、モデル設定、ファイルアクセスモード、出力 schema 参照を組み立てる責務を持つ。

## Read this when
- `cmoc review oracle` における oracle file レビューのサブタスク呼び出し設計を確認または変更したいとき。
- 新規所見列挙、所見の擁護理由列挙、反証理由列挙、人間提示の採否判定、所見リストの整理・マージのどの段階を読むべきか切り分けたいとき。
- レビュー用 AI に渡す既知所見、既知理由、対象所見、所見リストなどの補助入力が、各タスクのプロンプトへどう組み込まれるかを追いたいとき。
- レビュー各段階の構造化出力契約と、それを使う呼び出し側実装の対応関係を確認したいとき。

## Do not read this when
- oracle file そのものの正本仕様内容、oracle と realization の基本概念、または review oracle 標準の本文を理解したいとき。
- CLI 引数解析、レビュー結果の保存・表示・通知、返却された編集操作の適用など、レビュー用 AI 呼び出し後の処理を調べたいとき。
- 個別の oracle file を実際に探索して仕様矛盾や表記問題を判断するロジックを探しているとき。
- Structured Output schema の一般的な書き方や JSON Schema 検証ライブラリの使い方だけを確認したいとき。
