# `enumerate_finding.json`

## Summary
- 正本仕様断片レビューで、レビュー対象と関連仕様から見つかった新規所見だけを返すための結果仕様を定める。
- 所見がない場合は空の一覧で表すこと、各所見には重大度、短い見出し、主な根拠となる仕様断片、理由を含めることを求める。
- 重大度は、仕様断片間の明確な矛盾や実装者裁量で解消できない問題と、誤字・表記揺れなどの単純な表記問題を区別する。

## Read this when
- 正本仕様断片レビューの出力が、既知の関連所見と重複しない新規所見だけを列挙する形になっているか確認したいとき。
- レビュー結果に含める所見の重大度分類や、理由・根拠として必要な意味情報の境界を確認したいとき。
- 新規所見が存在しない場合のレビュー結果の扱いを確認したいとき。

## Do not read this when
- 正本仕様断片そのものの内容や、レビュー対象領域の仕様を理解したいとき。
- 既知の関連所見の管理方法、比較方法、探索方法を確認したいとき。
