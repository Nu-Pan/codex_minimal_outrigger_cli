# `resolve_parameter.json`

## Summary
- AI Agent CLI/TUI がオリジナルプロンプト実行前に、必要な論理ファイルアクセス権限と参照すべき標準文書群を判定して返すための構造を定義する JSON Schema。
- プロンプト解決処理における判定結果の形、必須性、許容される権限分類、各判定に理由説明を伴わせる制約を確認する入口になる。

## Read this when
- オリジナルプロンプトから実行時のファイルアクセス権限を分類する処理を実装・変更・検証するとき。
- プロンプト実行前に、oracle と realization の基本事項、各種標準、レビュー適用、INDEX.md エントリー生成に関する標準を読む必要があるかどうかを機械的に返す仕様を確認するとき。
- TUI/CLI のパラメータ解決結果を JSON Schema として検証したいとき。
- パラメータ解決結果に、判定値だけでなく判断理由も必ず含める必要があるかを確認するとき。

## Do not read this when
- 各標準文書そのものの要求内容を確認したいとき。対象は参照要否を判定するための構造であり、標準本文の代替ではない。
- oracle file や realization file の定義、編集責務、品質基準を直接確認したいとき。該当する正本仕様や標準本文を読む。
- レビューやレビュー適用や INDEX.md エントリー生成の具体的な作業手順を知りたいとき。対象はそれらの作業を実行するかどうかの判定結果形式を定めるだけである。
- 実装コードの現在の挙動や UI 表示の詳細を調べたいとき。対象は実装ではなく、パラメータ解決結果の検証構造である。

# `resolve_parameter.py`

## Summary
- AI Agent CLI/TUI に渡す実行パラメータを選定するためのエージェント呼び出し内容を構築する実装。
- 元プロンプトを埋め込み、選択可能なファイルアクセスモードと標準プロンプト群を含めた完全プロンプトを作り、効率重視モデル・中程度推論・読み取り専用アクセス・対応する structured output 保存先を指定した呼び出しパラメータを返す。

## Read this when
- TUI で入力された元プロンプトから、実行前にモデル種別・推論強度・ファイルアクセスモード・出力先をどう決めるかを確認したいとき。
- 実行パラメータ選定用エージェントに渡すプロンプトの role、summary、goal、補助情報、標準プロンプト有効化範囲を変更・調査するとき。
- TUI 実行で選択候補として提示するファイルアクセスモードの集合を確認・変更するとき。

## Do not read this when
- 実際のファイルアクセスモードごとの規則文そのものを確認したいだけのとき。
- 完全プロンプトを組み立てる共通処理や markdown レンダリングの詳細を調べたいとき。
- リポジトリルートや作業ルートの解決規則を確認したいとき。
- TUI の画面操作、入力編集、サブコマンド起動フローを調べたいとき。
