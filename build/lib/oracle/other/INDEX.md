# `cmoc_config.py`

## Summary
- cmoc のリポジトリ別設定を表すデータ構造を定義する。永続化先、JSON 化時の Enum value 化、初期化コマンドによる生成・同期、人間による調整対象であることを示す正本仕様断片である。
- Codex CLI 向けのモデル名・reasoning effort 名の対応、ファイルアクセス規則違反時のリカバリ試行回数、apply fork と review oracle の各ループ上限など、サブコマンド別の既定設定値を確認する入口になる。

## Read this when
- cmoc の設定項目、既定値、永続化形式、またはリポジトリごとの差し替え可能な挙動を確認したいとき。
- 設定ファイルの生成・同期・手編集の責務境界を確認したいとき。
- Codex CLI 呼び出し、apply fork、review oracle の挙動設定に関する実装やテストを変更するとき。
- Enum 系の値を設定 JSON に保存する際の表現を確認したいとき。

## Do not read this when
- パス概念、work/repo/run/root の定義だけを確認したいとき。
- 設定値ではなく、各サブコマンドの処理手順や入出力仕様そのものを確認したいとき。
- 設定の読み書き処理、JSON 変換処理、または CLI コマンド実装の詳細を確認したいとき。

# `path_model.py`

## Summary
- cmoc で使うルートパスプレースホルダの種類と、プレースホルダ付きパス・絶対パスを相互に扱うための基準と実装を定義する。
- プレースホルダなし相対パスを禁止し、実パス解決、実パスからプレースホルダ表記への変換、各ルートの探索条件を扱う。

## Read this when
- cmoc 内でパスを絶対パスまたはプレースホルダ付き相対パスとして表記・解決する仕様や実装を確認したいとき。
- ルートパスプレースホルダの意味、追加、命名、対応する実ディレクトリの判定条件を確認したいとき。
- プレースホルダを含む入力パスの検証、実パスへの解決、実パスからプレースホルダ表記への変換に関わる変更をするとき。
- git worktree、main worktree、実行時 cwd の関係から、cmoc がどのルートを選ぶべきかを調べるとき。

## Do not read this when
- パス表記やルート解決に関係しない CLI コマンドの挙動、出力形式、状態管理を調べたいとき。
- 個別機能がどのファイルを読み書きするかだけを知りたいとき。
- 正本仕様断片と実装成果物の責務境界など、パスモデル以外の開発ルールを確認したいとき。

# `struct_doc.py`

## Read this when
- 構造化された自然言語文章を Markdown 文字列へ変換する処理を確認・変更したいとき。
- 見出し階層、コードブロック、本文文字列のレンダリング結果や空行整理の挙動を確認したいとき。
- triple quoted string で書いた本文から前後の空行や共通インデントを除く正規化処理を確認したいとき。

## Do not read this when
- Markdown ファイル全体の仕様や内容そのものを確認したいだけで、レンダリング用ヘルパーの挙動に関心がないとき。
- CLI コマンド、永続状態、パスモデル、外部入出力など、構造化文章の Markdown レンダリング以外を調べたいとき。
