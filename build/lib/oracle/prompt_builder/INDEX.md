## Read this when

- プレースホルダ対応表のキーに何を入れるか、値として文字列以外にパスを渡せるかを確認したいとき。

## Do not read this when

- 利用可能なプレースホルダの一覧や各プレースホルダの意味を確認したいとき。

# `complete_prompt.py`

## Summary
- agent call に渡す完全なプロンプトを、静的プロンプト、動的プロンプト、プレースホルダ定義の順に組み立てる実装。
- oracle、realization、review、index entry、file access、routing rule などの標準プロンプト片を、指定フラグと依存関係に応じて注入する入口を担う。
- 呼び出しごとの変動要素を後半に寄せ、固定要素を前半に安定配置することで、プロンプトキャッシュヒット率を意識した構成を作る。

## Read this when
- agent call 用の完全なプロンプト構築手順、静的プロンプトと動的プロンプトの並び順、またはプレースホルダ定義の合成方法を確認・変更したいとき。
- oracle standard、realization standard、review standard、index entry standard などの注入フラグ間の依存関係を確認・変更したいとき。
- file access rule や routing rule が完全プロンプトへどの順序で追加されるかを確認したいとき。

## Do not read this when
- 個々の標準プロンプト片の本文や文言を確認・変更したいだけのときは、それぞれの prompt parts 実装を読む。
- StructDoc 自体の構造や表示形式、PlaceholderMap や FileAccessMode の定義を確認したいときは、それらを定義する基本モジュールを読む。
- agent call の実行、送信、結果処理の挙動を調べたいときは、プロンプト構築ではなく呼び出し側の実装を読む。

# `parts`

## Summary
- oracle file、realization file、INDEX.md、ファイルアクセス、レビュー基準など、AI エージェント向けプロンプトに差し込む規範・基本説明の部品群を扱う。
- 各部品は構造化文書として、正本仕様断片の扱い、具体化ファイルの品質基準、ルーティング文書の書き方、レビュー所見の境界、読み書き規則などを組み立てる入口になる。

## Read this when
- エージェントに渡す規範プロンプトのうち、oracle/realization の責務境界、品質基準、レビュー基準、INDEX.md の扱い、ファイルアクセス規則に関する文面を確認または変更したいとき。
- レビューで所見に含めるべき不整合、未定義部分として扱うべき範囲、realization file 単体の致命的問題の境界を確認したいとき。
- INDEX.md エントリーやルーティング規則として、読む条件・読まなくてよい境界・本文根拠の扱いを定めるプロンプト部品を調整したいとき。
- 読み書きモードごとにエージェントへ提示される禁止対象や、プレースホルダを含むアクセス規則文書の組み立てを確認したいとき。

## Do not read this when
- 個別サブコマンドの実行経路、CLI 入出力、永続状態、path model、実際のファイルシステム権限や sandbox 制御の実装を調べたいとき。
- 構造化文書、標準データ型、プレースホルダ変換などの汎用データ構造や変換処理そのものを確認したいとき。
- oracle file や realization file の定義済み規範を読むだけで、プロンプト部品としての組み立て箇所を確認する必要がないとき。
- レビュー対象ファイルの探索、git 管理対象判定、パッチ生成、または実装修正手順を調べたいとき。

# `prompt_.py`

## Summary
- agent call に渡す完全なプロンプトを、役割・概要・完了条件・ファイルアクセス制限・任意追加パーツ・各種標準注入フラグから組み立てる実装を扱う。
- oracle/realization、review、index entry などの標準プロンプトを必要フラグに応じて静的パーツとして追加し、依存する基本情報が欠けないように注入フラグを補正する。
- プロンプトキャッシュ効率のため、静的プロンプトを先に固定順で並べ、呼び出しごとに変わる情報とプレースホルダー定義を後半に置く構成を担う。

## Read this when
- agent call 用の完全なプロンプト構築順、静的プロンプトと動的プロンプトの分離、またはプレースホルダー定義の組み立てを確認・変更したいとき。
- oracle standard、realization standard、review standard、index entry standard などの注入フラグ間の依存関係を確認・変更したいとき。
- ファイルアクセス規則やルーティング規則が、完全プロンプトのどの位置で追加されるかを確認したいとき。
- 任意の静的・動的プロンプトを追加する入口や、追加パーツが最終プロンプトへ入る順序を確認したいとき。

## Do not read this when
- 個々の標準プロンプト本文そのものを確認・変更したいだけなら、それぞれの標準パーツを構築する対象へ進む。
- ファイルアクセス規則やルーティング規則の内容そのものを確認・変更したいだけなら、それらの規則パーツを構築する対象へ進む。
- 構造化ドキュメントの表現形式やプレースホルダー型の定義を確認したいだけなら、それらの基礎型を定義する対象へ進む。
- プロンプトを利用する呼び出し側の制御や実行フローを確認したいだけなら、呼び出し側の実装へ進む。
