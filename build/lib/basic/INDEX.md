# `acp.py`

## Summary
- AI コーディングエージェント呼び出し時のパラメータを表す基本型を定義する実装。論理モデルクラス、reasoning effort、ファイルアクセスモードの列挙と、プロンプト・Structured Output schema・一時的な追加書き込み許可パスを束ねる不変データクラスを扱う。

## Read this when
- エージェント呼び出しに渡す設定値の型、フィールド、既定値を確認したいとき。
- モデル選択、reasoning effort、ファイルアクセス権限を cmoc 内部でどの論理値として表すかを確認したいとき。
- Structured Output schema の有無や、通常の禁止範囲に対する呼び出し単位の追加書き込み許可を、呼び出しパラメータとしてどう保持するかを確認したいとき。

## Do not read this when
- バックエンドが受理する具体的なモデル名や reasoning effort 名への変換処理を確認したいとき。
- 実際のファイルアクセス制御、書き込み許可判定、サンドボックス適用の実装を確認したいとき。
- エージェント呼び出しの実行手順、プロセス起動、入出力処理、結果解析を確認したいとき。

# `path_model.py`

## Summary
- cmoc で使うルートトークン表記と実パス解決を扱うモジュール。絶対パス、ルートトークン付き相対パス、RootToken から現実の絶対パスへ変換し、逆に実パスを指定したルートトークン表記へ戻す責務を持つ。
- `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の意味と探索方法を定義し、`.git` ディレクトリ・`.git` ファイル・`bin/cmoc`・git common dir を手がかりに各 root を解決する。
- ルートトークンなし相対パスを拒否する入力検証、root 探索用の親ディレクトリ候補列挙、linked worktree と main worktree の関係を前提にした path 変換の入口になる。

## Read this when
- cmoc 内で `<cmoc-root>`、`<repo-root>`、`<run-root>`、`<work-root>` の定義や解決規則を確認したいとき。
- ルートトークン付きパスを実際の絶対パスへ変換する処理、または絶対パスをルートトークン表記へ戻す処理を変更・調査するとき。
- `.git` ディレクトリを持つ main worktree、`.git` ファイルを持つ linked worktree、ユーザーの cwd から見た最近傍 worktree root の区別が必要な path 関連処理を読むとき。
- ルートトークンなし相対パスをエラーにする挙動や、空パス・未知型・root 不一致時の例外条件を確認したいとき。

## Do not read this when
- 個別コマンドの CLI 引数、出力 schema、ユーザー向けメッセージを調べたいだけで、root token や実パス解決に触れないとき。
- INDEX.md の生成規則、oracle file と realization file の定義、ルーティング文書の記述方針を確認したいとき。
- 一般的なファイル読み書き、ディレクトリ走査、テスト fixture 管理など、cmoc 固有の root token 表記と無関係な path 操作を調べるとき。
- パス解決結果を利用する上位機能の挙動だけを確認したい場合で、その上位機能側のモジュールに直接進めるとき。

# `standard.py`

## Read this when
- 規範や standard の内部表現を確認したいとき。
- 背景・要求・判断例からなる規範を構造化ドキュメントへ変換する処理を確認または変更したいとき。
- 要求ラベルの許容値、必須フィールドの検証、判断例の扱いを確認したいとき。

## Do not read this when

- 特定の oracle file や realization file が従う具体的な規範本文を確認したいとき。

# `struct_doc.py`

## Summary

- 文書構造を表すクラスとレンダリング関数がまとまっており、Markdown 出力の整形規則を確認する入口になる。

## Read this when
- 構造化された見出し階層から Markdown 文字列を生成する処理を確認・変更したいとき。
- 文書本文やコードブロックのインデント正規化、空行圧縮、コードフェンス内の行保持に関する挙動を確認したいとき。
- Structured Output や説明文など、プログラム内で組み立てた自然言語文書を Markdown として出力する箇所の実装根拠を探すとき。
