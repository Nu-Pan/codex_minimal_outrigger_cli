# cmoc 自己呼び出し

- 非呼び出し側は cmoc についての概念を注入されていないはずなので、自己呼び出しは可能なはず
- 作業途中で壊れたら詰むので caller, callee は分ける必要がある
- `~/codex_minimal_outrigger_cli`
    - こっちが callee
    - `oracles` 書いたりとかはこっちでやる
- `~/codex_minimal_outrigger_cli_caller`
    - こっちが caller
    - こっちにパス通す
    - callee 側でマージが入ったらこっちで最新を引く

# oracle 全チェック定型文

自己開発が可能になるまではこれ使う
```markdown
/goal `<cmoc-root>/oracles` 配下の正本仕様断片を書き換えました。致命的な問題点が無いか評価してください。
ここで言う致命的な問題点とは、実装を参照せずに、仕様だけから判断・実装したときに、以下の問題が発生しうるものを「致命的な問題」と定義します。

- 主要ワークフローが壊れる
- 完了判定できない
- cmoc の中核目的を満たしたと判定できない
```
